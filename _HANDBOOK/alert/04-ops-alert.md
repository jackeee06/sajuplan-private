# 사장님이 받는 운영 알림 (OpsAlert)

## 한 줄로 답하면
**사주플랜 운영 중 중요 사건이 생기면 사장님에게 알림톡으로 알림. 결제 사고 / m2net 사고 / 자동 부재중 / 일일 요약 등 약 28가지.**

## 주요 알림 종류

### 결제·정산
- 결제 발생 / 실패
- m2net 잔액 동기화 실패
- PG (AG9) callback 실패
- 정산 완료
- 선지급 신청 / 입금 / 반려
- 고객보호비용 처리

### 채팅·통화
- 3분 자동 취소
- 상담사 자동 부재중
- 5초 환불 (G 정책) 발생

### 시스템
- 매일 09:00 일일 요약
- 외부 의존성 다운 (DB, m2net, BizM)
- cron 잡 실패
- DB 연결 사고

### 회원 운영
- 가입 발생 (옵션)
- 상담사 신청 대기

## 이런 상황을 보셨나요?

### 사장님이 "운영 알림 너무 많이 와요"

→ info 단계 알림이 많음. critical / warning 만 받기로 정책 조정 가능.

### "매일 일일 요약 안 와요"

→ 가능 원인:
1. 사장님이 채팅 중일 때 발송 (자동 차단)
2. cron 정지
3. BizM 발송 사고

→ 다음 날 자동 재발송 X. 못 받으면 사이드바 → 대시보드 에서 직접 확인.

### "사고 알림이 오는데 어떻게 처리해야 할지 모르겠어요"

→ [사고 매뉴얼](system/03-incident) 참조. 알림 종류별 대응 가이드.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 매일 09:00 일일 요약 도착 | ✅ 정상 |
| 결제 실패 알림 (다수) | ⚠️ 패턴 확인. AG9 사고 가능 |
| 사장님이 채팅 중일 때 운영 알림 차단 | ✅ 정책 (채팅 보호) |
| critical 알림인데 안 옴 | ❌ **사고**. 즉시 알림 시스템 점검 |
| 알림 받은 후 처리 안 함 | ⚠️ 사고 누적 위험 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 운영 알림 카탈로그 (28종) | 사이드바 → 알림 → 알림 가이드 |
| 사고 매뉴얼 | 운영 바이블 → 사고 매뉴얼 |
| 대시보드 (운영 상태 종합) | 사이드바 → 대시보드 |
