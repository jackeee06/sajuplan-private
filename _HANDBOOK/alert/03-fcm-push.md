# FCM 푸시는 어떻게 동작해요?

## 한 줄로 답하면
**Firebase 클라우드 메시지 기반 모바일 푸시. 인프라 완성 상태인데 거의 안 씀 (사장님 + 상담사들 알림톡 선호).**

## 현재 사용도

| 항목 | 상태 |
|---|---|
| 시스템 구축 | ✅ 완성 |
| 토큰 등록 | ✅ 동작 |
| 발송 API | ✅ 동작 |
| 실제 사용 | 거의 안 씀 (알림톡으로 통일) |

→ 회원이 푸시 받으려면 사주플랜 앱 설치 + 알림 권한 허용 필요. 카톡 알림톡이 도달률 ↑.

## 이런 상황을 보셨나요?

### "푸시 알림 안 와요" 라고 회원이 클레임

→ 가능 원인:
1. 회원 폰 OS 알림 권한 거부
2. 회원이 사주플랜 앱 미설치 (웹만 사용)
3. 푸시 시스템 일시 사고

→ 답변: "사주플랜은 알림톡이 메인 채널이에요. 푸시 못 받으셔도 알림톡으로 받으시면 됩니다."

### "푸시 켜고 싶어요"

→ 회원 마이페이지 → 알림 설정 → 푸시 ON (옵션 있다면). 또는 폰 OS 알림 권한 확인.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 사주플랜 앱에서 푸시 정상 동작 | ✅ 정상 |
| FCM 키 미설정 → 푸시 비활성 (개발 모드) | ✅ 정상 |
| 푸시 클릭 → 앱 열리며 특정 페이지 이동 | ✅ 정상 (deep link) |
| 푸시 발송했는데 실패율 높음 | ⚠️ 토큰 만료 다수. 정리 cron 검토 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 푸시 발송 | 사이드바 → 알림 → 푸시 알림 |
| 푸시 가이드 | 사이드바 → 알림 → 푸시 가이드 |
