# FCM 푸시 시스템 완전 가이드

## 한 줄로 답하면
**앱이 꺼져 있어도 상담사 폰에 도달하는 알림 시스템. 알림톡(카카오)이 외부 도달 메인이고, FCM 푸시는 앱 설치 사용자 대상 백업 채널이다. 2026-06-08 인프라 정비로 채팅/전화 요청 등 핵심 이벤트에 FCM 푸시 추가 완료.**

---

## 전체 구조 한눈에

```
회원이 채팅 신청
    ↓
서버 (NestJS)
    ├── 알림톡 (BizM) → 상담사 카카오톡 ← 메인 채널
    └── FCM 푸시 → Firebase 서버 → 상담사 폰 알림 ← 앱 꺼져도 도달
                                    ↓
                            앱 켜짐 (탭하면)
                            WebView가 해당 화면으로 자동 이동 ← 딥링크
```

## 현재 발송 이벤트 목록 (2026-06-08 기준)

| 이벤트 | 수신자 | 발송 방식 | 딥링크 |
|---|---|---|---|
| 채팅 상담 요청 | 해당 상담사 | 토큰 (1:1) | `/chat/{chatRoomId}` |
| 전화 상담 요청 | **전체 상담사** ⚠️ | 토픽 chl_5 | `/mypage` |
| 신규 문의(QnA) 도착 | **전체 상담사** ⚠️ | 토픽 chl_5 | `/counselor/mypage/customer-qnas/{id}` |
| 등급 승급 | 해당 상담사 | 토큰 (1:1) | `/counselor/mypage/grade` |
| 문의 신고 알림 | 신고당한 회원 | 토큰 (1:1) | `/mypage/my-qnas` |
| 관리자 일괄 발송 | 전체/회원/상담사 | 토픽 선택 | 없음 |

> ⚠️ **전화 요청·QnA 알림은 현재 전체 상담사 브로드캐스트 버그 있음** — chl_5 토픽 = 모든 상담사에게 다른 사람 알림이 감. 개선 필요.

## 앱 동작 방식

### 앱 상태별 동작

| 앱 상태 | FCM 동작 | 사용자 경험 |
|---|---|---|
| **포그라운드** (화면 켜짐) | Firebase SDK가 자동 알림 미표시 → 앱 내 인앱 배너로 표시 | 화면 상단에 배너 노출 |
| **백그라운드** (홈 내려감) | OS가 시스템 알림으로 표시 | 폰 알림창에 도착 |
| **종료** (완전히 닫음) | OS가 시스템 알림으로 표시 | 폰 알림창에 도착 |

### 알림 탭 → 딥링크 자동 이동 (✅ 구현 완료)

알림을 탭하면 앱이 열리며 **관련 화면으로 자동 이동**됩니다.

- 백그라운드 상태 탭 → `onNotificationOpenedApp` 핸들러
- 앱 종료 후 탭 → `getInitialNotification` 핸들러 (콜드스타트)
- 이동 경로: FCM data 페이로드의 `event_url` 또는 `link` 값

## 이런 상황을 보셨나요?

### "푸시가 안 왔어요"

확인 순서:
1. **앱 알림 권한** — 폰 설정 → 앱 → 사주플랜 → 알림 켜져 있는지
2. **앱 설치 여부** — 웹 브라우저로만 쓰는 사용자는 FCM 미지원
3. **FCM 서비스 계정 키 유효성** — PM2 로그에 `invalid_grant` 에러 확인 (키 갱신 필요)
4. **토픽 구독 여부** — 앱 로그인 이후 자동 구독. 재로그인으로 해결 가능

### "푸시는 왔는데 글자가 깨졌어요 (????? 표시)"

→ 푸시 발송 시 한글이 ASCII/cp949로 잘못 인코딩됨.
→ 서버 코드에서 발송하는 경우는 UTF-8 자동 처리되므로 정상.
→ 테스트용 스크립트(Python + PowerShell)에서 직접 발송할 때 Windows cp949 인코딩 문제 → UTF-8 파일로 작성 후 SFTP 전송으로 우회.

### "푸시 탭했는데 홈 화면만 떠요"

→ 딥링크 미구현 시 나타나던 현상. 2026-06-10 App.tsx에 `onNotificationOpen` 핸들러 추가로 해결. 최신 앱 빌드 설치 필요.

### "Firebase 오류가 나서 푸시가 전부 안 가요"

→ 서비스 계정 키 만료/삭제됨. PM2 로그에 `invalid_grant: Invalid JWT Signature` 에러 확인.
→ **해결**: Firebase 콘솔 → 프로젝트 `sajummon-5a4c0` → 설정 → 서비스 계정 → 새 키 발급 → JSON 파일 서버에 업로드.
→ 키 업로드 경로: `/data/wwwroot/api.sajumoon.co.kr/secrets/fcm-service-account.json`
→ 업로드 후 `pm2 reload sajumoon-api` 필요.

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 |
|---|---|
| 알림 탭 → 해당 채팅방/문의 페이지로 이동 | ✅ 정상 |
| 앱 꺼져 있는데 알림 도달 | ✅ 정상 |
| FCM 키 파일 없으면 푸시 비활성 (알림톡은 정상) | ✅ 정상 |
| 전화 요청 알림이 나 말고 다른 상담사에게도 감 | ⚠️ 알려진 버그 (chl_5 브로드캐스트) |
| 푸시 글자 깨짐 (?????로 표시) | ❌ 인코딩 문제 — 발송 코드 확인 |
| PM2 로그에 invalid_grant 반복 | ❌ Firebase 키 만료 — 즉시 갱신 필요 |
| 알림 탭해도 홈화면만 뜸 | ❌ 앱 구버전 사용 중 (딥링크 미포함 빌드) |
