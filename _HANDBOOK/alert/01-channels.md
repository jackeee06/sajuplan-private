# 사주플랜은 어떤 알림을 어떻게 보내나요?

## 한 줄로 답하면
**알림톡 (카톡, 외부 도달) + 인앱 알림 (사주플랜 앱 안) + 푸시 (FCM, 거의 안 씀) 3채널. 알림톡이 메인.**

## 3가지 채널

| 채널 | 도달 | 용도 |
|---|---|---|
| **알림톡** (BizM) | 카톡 외부 | 메인. 채팅 요청, 결제 확인, 정산 등 |
| **인앱 알림** | 사주플랜 앱 안 (종 모양) | 앱 진입자에게. 영구 보존 |
| **푸시** (FCM) | 폰 OS 알림 | 거의 안 씀 (상담사들이 알림톡 선호) |

## 정책

### 회원 / 상담사 입장
- 정보성·거래성 알림만 (가입, 결제, 채팅 요청, 정산 등)
- 마케팅성 알림 X (카카오 정책 + 사장님 결정)

### 채팅 중 차단
- 회원·상담사가 채팅 진행 중일 때 다른 알림톡 자동 차단
- 화이트리스트: `chat_request_to_counselor` (다른 회원의 새 채팅 요청) 만 통과

→ [채팅 도중 다른 알림 안 가는 이유](chat/04-alert-block)

## 이런 상황을 보셨나요?

### 회원이 "왜 답변 알림톡 안 왔어요?" 라고 클레임

→ 가능 원인:
1. 회원이 채팅 중이었음 (자동 차단)
2. 회원 폰이 070 차단
3. BizM 발송 사고

→ 답변: "마이페이지에 인앱 알림은 도착했습니다 (정보 손실 없음). 채팅 중이셨을 가능성이 큽니다."

### "사장님이 일일 요약 알림 못 받았어요"

→ 사장님이 그 시각에 채팅 중이었거나, 폰 차단. 일일 요약은 다음 날 자동 재발송 X (한 번 발송).

### "마케팅 알림톡 보내고 싶어요"

→ 카카오 정책상 거부될 가능성 큼. 정보성으로 우회 가능 (예: 이벤트 안내).

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 회원 답변 받음 → 알림톡 + 인앱 동시 | ✅ 정상 |
| 회원 채팅 중 → 알림톡 차단, 인앱은 발생 | ✅ 정상 (정책) |
| 알림톡 발송 후 인앱에는 안 옴 | ❌ **사고**. 인앱 알림 시스템 점검 |
| 마케팅 알림톡 시도 → BizM 거부 | ✅ 정상 (정책) |
| 사장님이 매일 일일 요약 못 받음 | ⚠️ 차단 또는 cron 사고. 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 알림톡 발송 이력 | 사이드바 → 알림 → 알림톡 발송 |
| 알림톡 템플릿 | 사이드바 → 알림 → 알림톡 템플릿 |
| 알림 가이드 (전체 알림 카탈로그) | 사이드바 → 알림 → 알림 가이드 |
| 푸시 발송 | 사이드바 → 알림 → 푸시 알림 |
