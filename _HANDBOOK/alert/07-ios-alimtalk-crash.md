# iOS 알림톡 클릭 시 앱 크래시 (OS 분기 임시조치)

## 한 줄로 답하면
**아이폰에서 알림톡 버튼을 누르면 사주플랜 앱이 꺼진다. 앱 재빌드 전까지, 백엔드에서 아이폰 사용자에겐 크래시 나는 알림톡을 안 보내고(푸시는 그대로), 상담신청만 버튼 없는 새 템플릿으로 받게 했다.**

## 무슨 일인가요?
- **아이폰** 사용자가 카카오 알림톡의 버튼(예: "답변내용 확인하기")을 누르면 → **앱이 검은 화면 후 강제 종료**.
- **안드로이드는 정상** (앱 켜지고 해당 페이지로 이동).
- 원인은 **아이폰 앱 내부**(네이티브) 문제. URL·백엔드·웹으로는 못 고침 → 앱 재빌드가 근본.

## 임시조치 (앱 재빌드 전까지, 2026-06-11~)
| 대상 | 처리 |
|---|---|
| **아이폰 회원** — 답변·후기·문의 알림톡 | **안 보냄(skip)**. 푸시로 받거나 앱 알림함에서 직접 확인 |
| **아이폰 상담사** — 상담신청(채팅·상담 요청) | **버튼 없는 새 템플릿**으로 받음(크래시 X) ※검수 통과 후 적용. 그 전엔 푸시로 |
| **안드로이드 전체** | 기존 그대로 (버튼 정상) |

→ 모든 경우 **FCM 푸시는 그대로 발송**되므로 상담사가 상담 요청을 놓치지 않음.

## 이건 정상인가요?
| 상황 | 답변 |
|---|---|
| 아이폰 회원에게 답변 알림톡이 안 옴 | ✅ 정상 (임시 skip). 앱 열면 알림함에 있음 |
| 아이폰 상담사가 상담신청을 푸시로만 받음 | ✅ 정상 (새 템플릿 검수 통과 전) |
| 안드로이드 알림톡 정상 동작 | ✅ 정상 |
| 아이폰에서 알림톡 버튼 눌러 앱 꺼짐 | ⚠️ 알려진 이슈. 앱 재빌드로 근본 해결 예정 |

## 근본 해결
iOS 앱의 `sajuplan://` 링크 처리(네이티브 openURL)를 고쳐 App Store 재배포 (Mac 빌드 필요). 완료되면 위 임시 분기를 제거하고 안드/iOS 통일.
