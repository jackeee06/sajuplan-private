# B-1 회원가입 / 로그인 / 비밀번호 찾기

## 한 줄로 답하면
**가입은 휴대폰 SMS 인증이 무조건 필요하고, 로그인은 틀렸을 때 "아이디가 없는지 비번이 틀린지" 절대 안 알려준다(보안). 비밀번호는 휴대폰 인증 후 직접 재설정하거나 이메일로 임시비번을 받는다.**

> 적대적 점검(2026-06-11)으로 아래 동작이 실제 코드와 일치함을 확인했습니다.

## 가입 (회원가입)

### 꼭 거쳐야 하는 단계
1. **아이디 중복확인** — [중복확인] 버튼을 눌러 사용 가능 여부 확인 (3~20자, 영문/숫자/_ 만)
2. **휴대폰 SMS 인증** — 번호 입력 → 인증번호 받기 → 인증 (이게 없으면 가입 자체가 안 됨)
3. 비밀번호 입력 (8~20자, 영문 + 숫자 각 1개 이상)
4. 닉네임·약관 동의 등 입력
5. m2net(통화·채팅 외부 시스템) 자동 등록 → 가입 완료

### 휴대폰 인증은 빠질 수 없다
- **인증을 안 했거나, 인증한 지 너무 오래(30분 초과)면 가입 거부** → `휴대폰 인증이 만료되었습니다. 인증번호를 다시 받아주세요.`
- 30분 여유를 두는 이유: 주소 검색·약관 정독 등 가입 폼 작성 시간을 충분히 포함하기 위함. 정상 사용자는 거의 안 걸림.

### 아이디 규칙 (중복확인 버튼이 검사)
- 3~20자, **영문/숫자/언더바(_)만**
- 소셜 가입 형식(`_K`/`_N` 으로 끝나는 ID)은 사용 금지 — 카카오/네이버 자동 ID와 헷갈리지 않게
- 관리자가 지정한 **금지 아이디**(admin 같은 예약어)는 차단
- 이미 쓰는 아이디면 사용 불가

### 비밀번호 규칙
- **8~20자** + **영문/숫자 각 1개 이상 필수** (2026-05-25 정책 강화)
- 너무 짧거나 영문/숫자 한쪽만 있으면 거부

### m2net 등록 실패 = 가입 통째로 취소
- 가입 마지막에 m2net(통화·채팅 라우팅 외부 시스템)에 회원을 등록한다.
- **여기서 실패하면 방금 만든 회원 정보까지 전부 롤백(취소)** 된다 → "반쪽 회원"이 안 생김.
- 즉 가입이 성공했으면 m2net 등록도 성공한 것.

## 로그인

### 가장 중요한 보안 규칙 — 실패 메시지를 하나로 통일
- **틀린 비밀번호든, 아예 없는 아이디든, 똑같은 메시지** 가 나온다:
  `가입된 회원아이디가 아니거나 비밀번호가 틀립니다.`
- 일부러 이렇게 한 것. "그 아이디는 있는데 비번만 틀렸다"는 식으로 알려주면 해커가 아이디를 캐낼 수 있어서다.
- 그래서 "내 아이디 있는지 확인하려고 로그인해봤는데 메시지가 똑같아요"는 **정상**.

### 차단/탈퇴 회원
- 탈퇴한 아이디 → `탈퇴한 아이디이므로 접근하실 수 없습니다.`
- 차단(접근금지)된 아이디 → `회원님의 아이디는 접근이 금지되어 있습니다.`

### 로그인 상태 유지
- 체크하면 14일간 자동 로그인 유지. 체크 해제 시 브라우저 닫으면 로그아웃.

## 비밀번호 찾기 (2가지)

### 1) 휴대폰으로 찾기 — 직접 새 비번 설정 (권장)
```
휴대폰 번호 입력 → 인증번호 전송 → 인증번호 입력 → 인증 완료
  ↓
바로 그 화면에서 새 비밀번호 직접 입력 → 변경 완료
```
- 인증 후 10분 안에 새 비번을 설정해야 함. 너무 오래 끌면 `휴대폰 인증이 만료되었습니다. 다시 인증해주세요.`
- 새 비번도 가입과 동일 규칙(8~20자, 영문+숫자).

### 2) 이메일로 찾기 — 임시비번 메일 발송
```
이메일 입력 → 임시비밀번호 자동 생성 → 메일로 발송
  ↓
임시비번으로 로그인 후 비번 변경 권장
```

### 소셜(카카오/네이버) 가입자는?
- 소셜 가입자는 비밀번호가 아예 없다 → 비번 찾기를 하면 `OO로 가입된 계정입니다. OO 로그인을 이용해주세요.` 안내.

### 인증 없이/틀린 인증번호로는 절대 안 됨
- 인증번호를 안 넣거나 틀리면 비번 찾기 자체가 차단(400 에러). "인증만 통과하면 누구 비번이든 바꾸는" 일이 없도록 막혀 있다.

## 이런 상황을 보셨나요?

### "인증번호가 안 와요"
→ 가능 원인:
1. 같은 번호로 **5분 안에 5회 초과** 발송 (한도 초과)
2. BizM(카카오 알림톡) 일시 사고
3. 번호 차단 설정
→ 답변: "5분 후 재시도 + 카카오톡 알림 설정 확인 부탁드립니다." (인증번호는 카카오톡으로 발송됨)

### "로그인 실패 메시지가 너무 두루뭉술해요"
→ 정상이자 의도된 보안. 아이디 존재 여부를 노출하지 않으려는 것.

### "비밀번호 찾기에서 인증했는데 새 비번 설정하려니 만료됐대요"
→ 인증 후 10분이 지나면 만료. 다시 인증번호를 받아 진행하면 됨.

### "카카오로 가입했는데 비밀번호 찾기가 안 돼요"
→ 정상. 소셜 가입자는 비번이 없어 카카오/네이버 로그인만 사용.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 인증 없이 가입 시도 → 거부 | ✅ 정상 (SMS 인증 필수) |
| 인증한 지 30분 넘어 가입 시도 → 거부 | ✅ 정상 (인증 만료 윈도우) |
| 비번 7자 또는 영문만 → 거부 | ✅ 정상 (8~20자 + 영문/숫자) |
| 없는 아이디·틀린 비번 → 같은 메시지 | ✅ 정상 (계정 존재 미노출) |
| 5분 안 인증번호 6번째 → 차단 | ✅ 정상 (어뷰징 차단) |
| m2net 등록 실패 → 가입 전체 롤백 | ✅ 정상 (반쪽 회원 방지) |
| 소셜 가입자 비번찾기 → 소셜 로그인 안내 | ✅ 정상 |
| 인증번호 없이 비번 재설정 시도 → 차단 | ✅ 정상 |
| 인증 보냈는데 안 오면서 비용은 빠짐 | ❌ **사고**. BizM/SMS 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 회원 가입 이력 | 사이드바 → 회원현황 → 고객 리스트 |
| 임시비번/인증 알림톡 발송 이력 | 사이드바 → 알림 → 알림톡 발송 |
| 가입 사고(m2net 등록 실패 등) | 고객 리스트 → 상세 / 시스템 로그 |
