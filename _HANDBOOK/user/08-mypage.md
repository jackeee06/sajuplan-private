# B-8 마이페이지 전체 탭 — 사용자 흐름

## 한 줄로 답하면
**`/mypage` 진입 시 로그인 상태면 회원 마이페이지(프로필·보유 코인·메뉴)가, 비로그인이면 환영 화면(로그인 버튼+공개 메뉴)이 뜬다. 그 아래 모든 탭(쿠폰/코인 내역/상담 내역/후기·문의/결제·충전/공지·이벤트·이용안내/앱 설정/상담사 신청)으로 갈라진다. 충전·앱설정처럼 개인정보가 필요한 탭은 비로그인으로 직접 주소를 쳐도 로그인 화면으로 보낸다.**

> 회원 ⇄ 상담사 모드 전환은 [member/04-dual-account](../member/04-dual-account) 참조. 이 문서는 **회원 마이페이지의 탭 구성과 진입 규칙** 관점.

---

## 진입점 — `/mypage` 한 주소, 두 화면

```
/mypage 접속
  ├─ 로그인 안 함        → 환영 화면 (MyPage)
  │     "사주플랜에 오신 걸 환영합니다" + 로그인 버튼 + 공개 메뉴 5개
  │
  └─ 로그인 함           → 회원 마이페이지 (MemberMyPage)
        프로필 + 보유 코인 박스 + 마이페이지 메뉴 + 추가메뉴 + 로그아웃
```

- 같은 `/mypage` 주소가 로그인 여부에 따라 다른 화면을 그린다. 비로그인이라고 에러가 나는 게 아니라 환영 화면이 정상 노출된다.
- 상담사 자격이 있는 사람도 **기본은 회원 마이페이지**를 본다. 우상단 핑크 `상담사 메뉴 >` 칩으로 상담사 영역으로 넘어간다.

---

## 로그인 사용자 마이페이지 구성

### 상단 — 프로필 + 보유 코인

- 프로필 영역: 아바타(클릭 시 회원 정보 수정), 닉네임, 인증 뱃지, 아이디. 우측에 `앱 설정` 칩(+상담사면 `상담사 메뉴` 칩).
- **보유 코인 박스**: 큰 핑크 숫자로 보유 코인을 표시하고, 아래 2분할 버튼 `코인 충전` / `결제내역`.
  - 이 화면은 **"코인"** 단어만 쓴다. "포인트"는 사용자 화면에 안 나온다.

### 마이페이지 메뉴 (5개)

| 메뉴 | 가는 곳 | 내용 |
|---|---|---|
| 쿠폰 | `/mypage/coupons` | 보유/사용 쿠폰, 쿠폰코드 등록 |
| 코인 내역 | `/mypage/points` | 코인 적립·사용 내역 + 보유 코인 |
| 상담 내역 | `/mypage/history` | 전화·채팅 상담 내역 |
| 나의 상담후기 | `/mypage/my-reviews` | 내가 쓴 후기 |
| 나의 상담문의 | `/mypage/my-qnas` | 내가 남긴 1:1 상담문의 |

### 추가메뉴 (5개 — 비로그인과 동일)

| 메뉴 | 가는 곳 |
|---|---|
| 이벤트 | `/mypage/events` |
| 이용안내 | `/mypage/help` |
| 공지사항 | `/mypage/notices` |
| 신규상담사 | `/mypage/new-counselors` |
| 상담사 신청 | `/mypage/counselor-apply` |

맨 아래 빨간 `로그아웃` (확인 모달 후 처리).

---

## 보호 라우트 — 비로그인 직접 접근 차단

개인정보·돈이 걸린 탭은 주소를 직접 쳐서 들어와도 **로그인 화면으로 보낸다**. 로그인 후 원래 보려던 화면으로 돌아온다.

| 탭 | 비로그인 직접 접근 시 |
|---|---|
| 코인 충전 (`/mypage/charge`) | `로그인` 화면으로 보냄 (redirect 보존) |
| 앱 설정 (`/mypage/app-settings`) | `로그인` 화면으로 보냄 |
| 회원 정보 수정 (`/mypage/member`·`/edit`) | `로그인` 화면으로 보냄 |
| 쿠폰 / 코인 내역 / 결제내역 | 서버가 로그인 필요(401) → 로그인 화면으로 보냄 |

반대로 **공개 탭**(이벤트·이용안내·공지사항·신규상담사·상담사 신청)은 비로그인도 그냥 본다.

---

## 코인 내역 화면 (`/mypage/points`)

- 상단에 **보유 코인** 카드 + `코인 충전` 버튼.
- 아래로 적립/사용 내역 리스트. 더보기 버튼으로 20건씩 추가 로드.
- **적립(+)은 핑크**, **사용(-)은 회색**으로 표시. 각 줄에 사유·시각·금액·사용 후 잔액이 함께 나온다.
- 표기는 전부 `코인` 단위 (예: `+30,000 코인`, `1,200 코인`).

---

## 코인 충전 화면 (`/mypage/charge`)

상단 3탭: **충전 / 사용내역 / 자동충전**.

- **충전 탭**: 결제요금(코인 패키지) 선택 → 결제방법(`사주플랜페이` / `일반결제`) → 결제하기.
  - 패키지에 `+%` 보너스 뱃지가 붙을 수 있다(예: 첫 결제 50% 추가 적립 안내).
  - `사주플랜페이`는 카드 등록이 있어야 즉시 결제. 카드가 없으면 카드 등록 화면으로 보낸다.
  - `일반결제`는 신용카드/가상계좌/페이코/카카오페이/네이버페이 중 선택 → PG 창으로 이동.
- **사용내역 탭**: 코인 내역과 같은 리스트를 충전 화면 안에서 바로 본다.
- **자동충전 탭**: 보유 코인이 기준(10,000) 아래로 내려가면 상담 중 자동 충전. 카드 등록 필요.

> 자동충전·사주플랜페이 등록은 [payment/07-auto-charge](../payment/07-auto-charge) 참고.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 비로그인 `/mypage` → 환영 화면 | ✅ 정상 (에러 아님) |
| 로그인 `/mypage` → 프로필+보유 코인 | ✅ 정상 |
| 비로그인이 `/mypage/charge` 주소 직접 입력 → 로그인 화면 | ✅ 정상 (보호 라우트) |
| 상담사 계정도 `/mypage`에서 회원 화면이 먼저 보임 | ✅ 정상 (`상담사 메뉴` 칩으로 전환) |
| 코인 내역에서 사용이 회색, 적립이 핑크 | ✅ 정상 |
| 충전 화면에 "포인트"라는 단어가 보임 | ⚠️ 용어 위반. "코인"으로 통일해야 함 |
| 결제내역에 진행중/실패 건이 안 보임 | ✅ 정상 (완료 건만 노출, DB엔 남아있음) |

## 자주 묻는 질문

### Q. 충전했는데 결제내역이 비어 있어요.
결제내역은 **결제 완료 건만** 보여줍니다. 가상계좌 입금 전(입금대기)은 24시간 이내일 때만 상단에 옅게 안내됩니다. 입금이 끝나면 완료 목록에 들어옵니다.

### Q. 상담사인데 충전 메뉴가 안 보여요.
상담사 메뉴 화면(보라 모드)에는 충전이 없습니다. 우상단으로 회원 모드로 돌아오면 보입니다. ([member/04-dual-account](../member/04-dual-account))

### Q. 비로그인인데 이벤트·공지는 왜 보이나요?
이벤트·이용안내·공지사항·신규상담사·상담사 신청은 **공개 메뉴**라 로그인 없이 봅니다. 충전·쿠폰·코인 내역만 로그인이 필요합니다.
