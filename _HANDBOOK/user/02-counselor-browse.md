# 상담사 목록 / 필터 / 검색 (browse)

## 한 줄로 답하면
**회원이 상담사를 찾는 3가지 경로 — ① 상담사 리스트(카테고리·분야·상담가능만 필터) ② 검색(이름·해시태그·분야) ③ 인기 검색어. 결과는 모두 상담사 카드로 보여주고, 지금 상담 가능한 사람이 위로 정렬된다.**

## 동작

```
[상담사 리스트]
 상단 4탭 (전체/사주/타로/신점)  ← 카테고리
  + 강조 칩줄 (⭐이벤트 상담사 / 💕재회)
  + 분야 칩줄 (#연애운 #재물운 … 가나다순)
  + "상담가능만 보기" 체크박스
   ↓
 카드 리스트 (상담중 → 대기(전화ON) → 대기(채팅전용) → 부재 순)
   ↓
 "더보기" 1번 클릭 = 전체 펼침

[검색]
 돋보기 → 검색 페이지 (인기 검색어 노출)
  → 검색어 입력 → 검색 결과 (이름·해시태그·분야·소개 부분일치)

[인기 검색어]
 실제 검색 로그(최근 7일) 상위 + 부족분은 상담사 해시태그로 채움
 + 관리자가 핀 고정한 검색어
```

## 정렬 규칙 (회원이 보는 순서)

리스트는 항상 "지금 상담할 수 있는 사람"이 위로 옵니다.

| 순서 | 상태 | 의미 |
|---|---|---|
| 1순위 | 상담중 (최근 5분 내) | "활발하게 상담 중" — 인기 연출 |
| 2순위 | 대기 + 전화상담 ON | 지금 전화 가능 |
| 3순위 | 대기 + 채팅 전용 | 지금 채팅 가능 |
| 4순위 | 부재 (ABSE/RESV) | 카드에 "상담요청하기" 버튼 노출 |

- **전체 탭**: 추천 상담사(is_recommended)가 맨 위, 그 다음 위 순서.
- **사주/타로/신점 탭**: 해당 카테고리만 거른 뒤 위 순서.
- 같은 순위 안에서는 최근 접속순(updated_at).

## 필터 동작 규칙

| 필터 | 동작 |
|---|---|
| 카테고리 4탭 (전체/사주/타로/신점) | 탭 누르면 백엔드 재조회. `member.counselor_category` 기준 |
| 분야 칩 (#연애운 등) | 카드의 해시태그에 포함되면 노출. 칩 다시 누르면 해제 |
| ⭐이벤트 상담사 칩 | 현재 진행 중인 이벤트 상담사만 노출 (최대 별도) |
| 💕재회 칩 | 항상 고정 노출되는 분야 칩 (재회 수요 큼) |
| 상담가능만 보기 | 지금 전화 또는 채팅이 즉시 가능한 상담사만 |

- **분야 칩 목록은 자동 생성**: 현재 카테고리 안 상담사들의 실제 해시태그에서 뽑아 가나다순 정렬. 큰 카테고리(사주/타로/신점)는 칩 줄에서 제외(중복 방지).
- **탭을 바꾸면** 선택했던 분야가 새 탭에 없으면 자동 해제됩니다.
- **필터를 바꾸면** "더보기"로 펼친 개수가 처음(10명)으로 리셋됩니다.

## 검색 동작

- 검색어가 **이름·닉네임·해시태그·전문분야·한줄소개·소개·약력** 중 하나라도 부분일치하면 결과에 포함.
- 정렬 우선순위: ① 이름/닉네임 정확일치 → ② 해시태그 정확일치 → ③ 이름 부분일치 → ④ 해시태그 부분일치 → ⑤ 분야/헤드라인 → ⑥ 소개/약력. 같은 순위는 후기 많은 순.
- 검색어가 비어 있으면 결과 없음.
- 검색하면 검색 통계(search_log)에 기록되어 인기 검색어에 반영됩니다.

## 인기 검색어 규칙

1. **실제 검색 로그 우선** — 최근 7일간 사용자가 많이 친 검색어 상위.
2. **부족하면 해시태그로 보충** — 활성 상담사 해시태그 빈도 상위로 빈 자리 채움.
3. **관리자 핀 고정** — 특정 순위에 고정한 검색어가 있으면 그 자리에 박힘.
4. **NEW 뱃지** — 검색어는 최근 24시간 내 처음 등장, 해시태그는 최근 30일 내 가입 상담사면 NEW.

## 카드에 보이는 정보

| 요소 | 내용 |
|---|---|
| 프로필 사진 | 좌하단에 카테고리 뱃지 (얼굴 가림 방지) |
| 뱃지 색 | 타로=핑크 / 신점=청록 / 사주=빨강 / 심리=보라 |
| **상담사 번호** | 이름 옆 `N번` (얇은 핑크 테두리 + 검은 숫자) — m2net 자동번호 + 150 |
| NEW 뱃지 | 가입 90일 이내 상담사 (이름 옆) |
| 단골 하트 ♡ | 누르면 단골 등록(핑크). 비로그인은 로그인 페이지로 |
| 가격 | "30초당 X,XXX원" |
| 버튼 | 전화상담 / 채팅상담 (부재면 "상담요청하기" 풀폭) |
| 후기 수 | 후기 1개 이상이면 "⭐후기(N)" |

> 별점(평균 평점)은 데이터 정착도가 낮아 **현재 카드에 노출하지 않음** (수집은 계속). 단가는 "30초당" 표기, 정산이 아니라 회원 차감 단가입니다.

### 상담사 번호 (2026-06-11)
경쟁사처럼 상담사별 번호를 **이름 옆**에 표시. 단골이 "153번 선생님"으로 기억·재방문하는 식별 번호입니다.
- **표시 번호 = m2net 자동번호(상담사연결번호) + 150** (자동번호가 1·4 처럼 너무 작아 보기 좋게 가산). 예: m2net `1번` → 화면 **`151번`**.
- **m2net 미등록 상담사(더미 `90001~`)는 번호 미표시** — 등록되면 자동으로 생김. 상담사 1000명 미만 정책 → 정상 번호는 1~999 → +150 = 151~1099.
- ⚠️ **표시만 +150. 전화/채팅 연결(m2net 연동)은 원본 번호 그대로** 사용 → 연결엔 전혀 영향 없음.
- 운영자 주의: 앱 화면 `153번` = m2net 화면 `상담사연결번호 3번` (즉 **−150** 해야 m2net 에서 그 상담사를 찾음).

## 이런 상황을 보셨나요?

### "신점 상담사인데 사주 탭에 떠요 / 안 떠요"
→ 분류 기준은 관리자가 설정한 `counselor_category`. 어드민에서 해당 상담사 카테고리 칩을 정확히 지정하면 즉시 반영. (옛 데이터는 해시태그 추정 fallback)

### "부재 상담사인데 왜 목록에 보여요?"
→ 정상. 2026-05-22부터 부재 상담사도 맨 아래에 노출하고 "상담요청하기" 버튼으로 회원이 호출 알림을 보낼 수 있게 했습니다.

### "검색했는데 분명 있는 상담사가 안 나와요"
→ 한 글자 검색어이거나 한글 자모만 친 경우(예: "ㅅㅏ") 통계에서 걸러집니다. 검색 자체는 이름·해시태그·소개까지 보므로 완성된 단어로 재검색 안내.

### "인기 검색어가 항상 같은 것만 떠요"
→ 실제 검색이 적으면 해시태그로 채워져서 그렇습니다. 검색이 쌓이면 자연스럽게 실제 검색어가 올라옵니다. 강조하고 싶은 단어는 관리자 핀 고정 가능.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 상담중 상담사가 리스트 맨 위 | ✅ 정상 (인기 연출 정책) |
| 부재 상담사가 맨 아래 + "상담요청하기" | ✅ 정상 |
| 탭 바꾸니 선택한 분야 칩이 풀림 | ✅ 정상 (새 탭에 없는 분야 자동 해제) |
| 본인(상담사)이 본인 카드 봄 | ✅ 정상 (2026-05-22 노출 정책) |
| "더보기" 1번에 전부 펼쳐짐 | ✅ 정상 (페이지네이션 → 누적 더보기) |
| 차단당한 회원에게 그 상담사 안 보임 | ✅ 정상 (블랙리스트 숨김) |
