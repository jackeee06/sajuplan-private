# B-4 단골 (즐겨찾기) — 사용자 흐름

## 한 줄로 답하면
**회원이 상담사 하트를 눌러 단골에 추가하고, 단골 페이지에서 모아 보고, 다시 눌러 해제하는 흐름. 같은 상담사를 두 번 눌러도 중복 안 되고(멱등), 비로그인이면 로그인 안내가 뜨며, 본인은 본인을 단골 등록할 수 없다.**

> DB 구조·운영 SQL·인기 상담사 통계 등 운영자/AI 레퍼런스는 [member/05-favorites](../member/05-favorites) 와 그 `.tech` 문서를 본다. 이 문서는 **사용자가 겪는 흐름** 관점.

---

## 사용자가 겪는 3가지 흐름

### 1) 단골 추가

```
상담사 카드 / 상담사 상세 → 하트 아이콘 클릭
  ↓
하트가 핑크색으로 바뀌고 "단골에 추가되었습니다" 토스트
  ↓
이후 단골 페이지·홈 배너에 반영
```

- 추가 직후 해당 상담사의 **팬 수(fan_count)** 가 즉시 1 증가해 화면에 반영된다.
- 추가는 **로그인한 회원만** 가능.

### 2) 단골 조회

```
마이페이지 / 단골 페이지 진입
  ↓
내가 등록한 단골 상담사 목록 (등록 최신순)
  ↓
카테고리(전체/사주/타로/신점) 칩으로 필터 가능
```

- 정렬: **단골 등록이 최근인 상담사가 위**.
- 떠난(탈퇴) 상담사는 목록에서 자동 제외된다.

### 3) 단골 해제

```
단골 페이지(또는 카드)에서 핑크 하트 다시 클릭
  ↓
"단골에서 해제되었습니다" 토스트 → 목록에서 사라짐
```

- 해제 후 팬 수는 즉시 1 감소.

---

## 꼭 알아야 할 동작 규칙 (2026-06-11 점검)

### 멱등성 — 두 번 눌러도 안전

- 이미 단골인 상담사를 또 추가해도 **중복 등록되지 않고** 정상 처리된다.
  (하트 더블클릭, 느린 네트워크 재시도 모두 안전. 에러 화면 안 뜸.)
- 없는 단골을 해제해도 에러 없이 넘어간다.

### 비로그인 차단

- 로그인하지 않은 상태에서 하트를 누르면 **로그인 안내 모달**이 먼저 뜬다.
  "로그인" 클릭 시 로그인 화면으로 이동하고, 보던 위치는 보존된다.
- 즉 비로그인은 단골 추가·조회 자체가 불가(서버가 401 반환).

### 본인 단골 등록 차단 ⚠️

- 회원⇄상담사 **듀얼계정** 사용자가 자기 자신을 단골 등록하려 하면
  **"본인을 단골 등록할 수 없습니다" (403)** 로 막힌다.
- 과거엔 이 차단이 무력화되어 본인을 단골 등록할 수 있었으나
  **2026-06-11 수정**으로 정상 차단된다 (원인은 .tech 문서 참조 — JWT 타입 비교 버그).
- 상담사가 자기 카드를 리스트/검색에서 **보는 것**은 허용(미리보기). 단골 **등록 액션**만 막힌다.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 하트 클릭 → 핑크 + "단골에 추가되었습니다" | ✅ 정상 |
| 핑크 하트 재클릭 → "단골에서 해제되었습니다" | ✅ 정상 |
| 이미 단골인 상담사 또 추가 | ✅ 정상 (멱등 — 중복 안 생김) |
| 비로그인 하트 클릭 → 로그인 모달 | ✅ 정상 |
| 본인이 본인 단골 등록 → 차단 메시지 | ✅ 정상 (403) |
| 단골 추가/해제 후 팬 수 즉시 변동 | ✅ 정상 (실시간 집계) |
| 단골 해제 후에도 페이지에 남음 | ⚠️ 캐시. 새로고침 |

## 자주 묻는 질문

### Q. 단골 한도가 있나요?
무제한입니다. 다만 많아지면 단골 페이지가 길어집니다.

### Q. 단골 상담사가 접속하면 알 수 있나요?
홈 진입 시 "단골 N명 접속중" 배너로 안내됩니다(접속중인 단골만, 최대 5명 노출). 푸시가 아니라 앱에 들어왔을 때 보여주는 방식입니다.
