# 채팅 시작부터 끝까지 — 어떤 단계를 거치나요?

## 한 줄로 답하면
**채팅은 [대기 → 진행 → 종료] 3단계를 지나가며, 사주플랜이 각 단계에서 다른 처리를 합니다.**

## 채팅 3단계

### 1단계 — 대기 (회원만 들어와 있음)
- 회원이 채팅 신청 → 상담사 알림톡 발송 → 회원은 대기 화면
- **코인 차감 0** (F 정책)
- 회원이 이때 종료해도 손해 없음

### 2단계 — 진행 (상담사 입장 후 채팅 중)
- 상담사가 입장하면 채팅이 시작됨
- **선택한 시간만큼 코인 차감 시작** (예: 15분치)
- 회원·상담사가 메시지 주고받음

### 3단계 — 종료
- 명시적 종료 (종료 버튼)
- 시간 소진 (선택한 시간 다 씀)
- 자동 취소 (3분간 무응답)
- 5초 자동 환불 (상담사 입장 직후 끝낸 경우)

## 이런 상황을 보셨나요?

### "회원이 채팅 신청만 하고 안 들어왔는데 어떻게 되나요?"

→ **사주플랜이 3분 기다린 후 자동 취소합니다.** 그 사이 상담사가 입장 안 하면 자동 정리되어 양쪽에 알림톡 발송.

### "상담사가 채팅방에 입장했는데 회원이 안 보여요"

→ 회원이 잠시 다른 앱 본 상태일 수 있어요. 사주플랜은 회원이 "잠시 자리비움" 표시 중. 3분 안에 돌아오면 그대로 진행.

### "채팅 끝났는데 다시 보고 싶다고 회원이 클레임"

→ 종료된 채팅은 마이페이지에서 다시 볼 수 있습니다. 메시지는 사주플랜이 보관해요. 회원에게 안내.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 회원 신청 후 즉시 대기 상태 | ✅ 정상 (1단계) |
| 상담사 입장 후 채팅 시작 | ✅ 정상 (2단계) |
| 3분 무응답 → 자동 취소 + 알림톡 | ✅ 정상 (3단계 자동) |
| 대기 상태가 4분 넘게 유지됨 | ❌ **사고**. 자동 취소 시스템 점검 필요 |
| 끝난 채팅인데 다시 진행 상태로 보임 | ❌ **사고**. 종료는 단방향이라 되돌아갈 수 없음 |
| 진행 중인데 코인이 0인 채로 계속 차감 | ❌ **사고**. 정산 시스템 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 진행 중인 채팅 | 사이드바 → 상담관리 → 채팅내역 리스트 → 필터 "진행중" |
| 종료된 채팅 다시 보기 | 사이드바 → 상담관리 → 채팅내역 리스트 → 채팅방 클릭 |
| 자동 취소된 채팅 | 사이드바 → 상담관리 → 채팅내역 리스트 → 종료 사유 컬럼 |
