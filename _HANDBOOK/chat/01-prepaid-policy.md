# 채팅 코인 차감은 언제 일어나나요? (선결제 정책)

## 한 줄로 답하면
**회원이 선택한 시간 (15·30·45·60분) 만큼 채팅 시작 시점에 코인이 빠집니다. 단, 상담사가 입장하기 전엔 절대 안 빠지고, 5초 안에 끝나면 자동 환불됩니다.**

## 이런 상황을 보셨나요?

### 회원이 "15분 채팅했는데 10분만 쓰고 끝났어요. 5분 환불해주세요" 라고 클레임

→ **환불 안 합니다.** 사주플랜 정책은 **선택한 시간 전체 차감** 입니다. 10분 쓰고 끝났어도 15분치 차감 유지. 회원이 시작 전 안내 화면에서 동의한 정책이에요.

→ 답변: "선택하신 15분치 차감은 정상이며, 사용 시간과 무관합니다. 다음에는 짧은 옵션 (15분) 으로 시작해보세요."

### 회원이 "상담사가 안 들어왔는데 코인이 빠졌어요" 라고 클레임

→ **사고입니다.** 사주플랜은 **상담사가 채팅방에 입장하기 전까지 절대 코인을 빼지 않습니다 (F 정책)**. 만약 빠졌다면 즉시 사장님께 보고 + 점검 필요.

→ 답변 (회원에게): "확인 후 즉시 정정 처리하겠습니다." 그 후 사이드바 → 매출현황 → 환불 이력 에서 수동 환불.

### 회원이 "채팅 시작하자마자 상담사가 인사하고 바로 끊었어요. 환불해주세요" 라고 클레임

→ **5초 안에 끝났다면 자동 환불 (G 정책).** 사주플랜이 자동으로 처리합니다. 만약 5초 이상 사용했다면 환불 X.

→ 답변: "채팅 시간을 확인해보겠습니다. 5초 이내면 자동 환불, 그 외엔 사용한 것으로 간주됩니다."

## 사주플랜의 2가지 회원 보호 정책

### F 정책 — 상담사 입장 전엔 차감 0
- 회원이 채팅 신청 → 상담사 응답 대기
- 이 단계에서 회원이 종료하든 시간 지나든 **코인 차감 0**
- 회원이 안심하고 신청 가능

### G 정책 — 5초 이내 자동 환불
- 상담사가 입장 후 채팅 시작
- 5초 안에 회원 (또는 상담사) 가 종료하면 **차감된 코인 전액 자동 환불**
- 한도: **하루 2회, 일주일 4회** (어뷰징 차단)

## 회원이 자주 묻는 질문

### Q. "왜 15분 다 안 썼는데 환불 안 되나요?"

선결제 정책이라서 그래요. 회원이 15분을 선택했다는 건 "15분치를 미리 결제" 한다는 의미. 음식점에서 정식 시켰는데 절반 먹고 환불해달라는 것과 비슷해요.

→ 답변: "이게 정책이라서 환불은 어렵습니다. 다음번엔 더 짧은 옵션 (15분 등) 으로 시작해보세요."

### Q. "5초 환불을 매일 받을 수 있나요?"

아니요. 하루 2회, 일주일 4회 한도가 있어요. 그 이상은 자동 환불 안 됩니다.

→ 답변: "5초 환불은 실수 보호 정책이라 한도가 있습니다. 한도 초과 시 사용한 것으로 간주됩니다."

### Q. "상담사가 30분 약속했는데 15분만 채팅했어요"

회원이 선택한 시간이 15분이면 15분치만 빠집니다. 30분이 필요했다면 30분 옵션을 선택해야 했어요. **사주플랜이 임의로 시간을 늘리거나 줄이지 않습니다.**

## 이건 정상인가요? 에러인가요?

| 회원/상담사 클레임 | 답변 |
|---|---|
| "15분 신청 → 5분만 쓰고 끝났는데 환불해줘요" | ✅ 정상 정책. 환불 거부. |
| "상담사 안 들어왔는데 코인 빠졌어요" | ❌ **사고**. F 정책 위반. 점검 필요. |
| "상담사 입장 3초 후 끊겼는데 환불 안 됨" | ⚠️ G 정책 한도 도달했을 가능성. 회원의 환불 한도 확인. |
| "같은 회원이 일주일에 5번 5초 환불 받음" | ❌ 한도 위반. G 정책상 주 4회까지만. 어뷰징 의심. |
| "회원 잔액이 사주플랜과 m2net 측에서 다름" | ⚠️ 동기화 어긋남. 자동 정정 안 되면 [m2net 관계](payment/01-m2net-relation) 참조. |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 채팅 사용 이력 | 사이드바 → 매출현황 → 사용(상담) 내역 |
| 환불 이력 | 사이드바 → 매출현황 → 환불 이력 |
| G 정책 5초 환불 받은 회원 | 사이드바 → 매출현황 → 환불 이력 → 사유 컬럼 "G 환불" |
| 회원 잔액 정정 (수동) | 사이드바 → 매출현황 → 포인트 관리 |

## 정책을 바꾸고 싶다면

- **선택 시간을 사용 시간만큼만 차감 (분당 차감 모델)** → 권장 X. 옛 정책으로 사고 빈발해서 선결제로 통일.
- **G 정책 5초 → 더 길게 (예: 30초)** → 어뷰징 위험 ↑. 변경 시 한도 (일/주) 도 같이 검토.
- **G 정책 한도 변경** → 어뷰징 우려 + 정상 사용자 보호 균형 고려.

변경 결정 시 사장님께 보고 후 시스템 수정 필요.
