# 운영 통계 / KPI 어디서 봐요?

## 한 줄로 답하면
**사이드바 → 대시보드 에서 매출·신규 가입·활성 채팅 등 종합 보기. 사이드바 → 통계 / 운영 KPI 에서 누계 통계.**

## 주요 KPI

| 지표 | 의미 |
|---|---|
| 일/월 매출 | 결제 완료 코인 합 |
| MAU | 월 활성 회원 (로그인 또는 채팅·통화 사용) |
| 신규 가입 | 일별 / 월별 |
| 정산 합계 | 상담사 정산 누적 |
| 환불 비율 | 환불 / 매출 |
| 상담사 가동률 | 활성 상담사 / 전체 |
| 채팅·통화 시간 | 일별 누적 |

## 현재 단계 (테스트)

현재 사주플랜은 **테스트 단계**라 실측 통계 의미 작음. 정식 운영 시작 후 1-2개월 후부터 의미 있는 데이터.

## 이런 상황을 보셨나요?

### "어제 매출 얼마야?" — 사장님 자주 묻는 질문

→ 사이드바 → 대시보드 → 일 매출 카드. 또는 매일 09:00 일일 요약 알림톡.

### "특정 회원이 한 달간 얼마 썼어?"

→ 사이드바 → 회원현황 → 고객 리스트 → 회원 검색 → 상세 → 결제 / 사용 이력.

### "이 상담사 한 달 매출은?"

→ 사이드바 → 회원현황 → 상담사 리스트 → 상세 → 정산 / 사용 내역.

### "환불 비율이 높은 것 같아"

→ 사이드바 → 매출현황 → 환불 이력 → 기간 필터 + 통계.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 일 매출 1만원 (테스트 단계) | ⚠️ 테스트라 정상 |
| MAU 매우 낮음 | ✅ 테스트 단계 정상 |
| 환불 비율 50%+ | ⚠️ 운영 시 비정상. 어뷰징 또는 사고 |
| 정산 합 > 매출 | ❌ 사고. 데이터 검증 |
| KPI 페이지 무한 로딩 | ❌ DB 쿼리 사고. 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 대시보드 (종합) | 사이드바 → 대시보드 |
| 운영 KPI | 사이드바 → 매출현황 → 운영 KPI |
| 통계 (상세) | 사이드바 → 통계 |
| 영업이익 시뮬레이터 (슈퍼만) | 사이드바 → 영업이익 시뮬레이터 |
