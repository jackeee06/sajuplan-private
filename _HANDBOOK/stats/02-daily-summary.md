# 매일 09:00 운영 요약 알림

## 한 줄로 답하면
**사주플랜이 매일 아침 09:00 (KST) 사장님에게 전일 매출·신규 가입·채팅 등 요약을 알림톡으로 발송.**

## 받는 내용 (예시)

```
[사주플랜 운영 알림]
유형: 일일 요약
시각: 2026-05-31 09:00
- 매출: 1,250,000원
- 신규 가입: 12명
- 채팅: 47건 (총 322분)
- 통화: 8건 (총 41분)
- 환불: 3건 (45,000원)
- 정산 대기: 8,500,000원
```

## 이런 상황을 보셨나요?

### "오늘 일일 요약 알림 안 왔어요"

→ 가능 원인:
1. 사장님이 09:00 시점에 채팅 중 (자동 차단 정책)
2. cron 정지
3. BizM 발송 사고

→ 답변: "사이드바 → 대시보드 에서 직접 확인 가능합니다."

### "수치가 사이드바 대시보드와 달라요"

→ 일일 요약 = 전일 (어제) 통계. 대시보드 = 실시간 + 일별. 시점 차이.

### "발송 시각을 다른 때로 바꾸고 싶어요"

→ 시스템 cron 설정 변경 필요. 사장님 결정 후 코드 수정.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 매일 09:00 알림 도착 | ✅ 정상 |
| 채팅 중일 때 안 옴 | ✅ 정책 (차단) |
| 수치가 0 또는 매우 낮음 | ⚠️ 테스트 단계 정상 |
| 매일 같은 수치 (어제 vs 오늘) | ❌ **사고**. 집계 SQL 의 날짜 조건 오류 |
| 알림 안 오는 날이 잦음 | ❌ cron 정지 또는 BizM 사고. 점검 |
