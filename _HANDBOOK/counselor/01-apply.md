# 상담사 신청은 어떻게 처리되나요?

## 한 줄로 답하면
**회원이 신청 → 운영자가 검토 → 승인 시 상담사 권한 + m2net 자동 등록. 회원 역할은 유지 (듀얼 계정).**

## 신청 흐름

```
회원 (마이페이지 → 상담사 신청)
  ↓
입력: 닉네임, 소개글, 카테고리 (사주/타로/신점), 프로필 이미지
  ↓
신청 접수 → 운영자에게 알림
  ↓
운영자 검토 (사이드바 → 회원현황 → 상담사 신청 내역)
  ↓
승인 → 상담사 권한 부여 + m2net 등록 + 환영 알림톡
반려 → 사유 입력 → 회원에게 안내 알림톡
```

## 이런 상황을 보셨나요?

### "상담사 신청한 회원 검토하고 싶어요"

→ 사이드바 → 회원현황 → 상담사 신청 내역 → 신청자 상세 → 입력 정보 확인 → 승인 또는 반려.

### "승인했는데 회원이 상담사 모드 진입 못 해요"

→ m2net 등록 실패 가능. 사이드바 → 회원현황 → 상담사 리스트 → 상세 → csrid 확인. NULL 이면 수동 재등록 필요.

### "반려했는데 같은 회원이 또 신청해요"

→ 정상. 재신청 가능. 사유 보고 재검토.

## 검토 시 확인할 점

- ✓ 프로필 이미지 적절한가? (선정적·부적절 X)
- ✓ 소개글 작성 됐는가?
- ✓ 카테고리 명확한가?
- ✓ 옛 어뷰징 이력 없는가? (회원 활동 검토)

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 신청 → 운영자 검토 대기 (pending) | ✅ 정상 |
| 승인 → 상담사 권한 + csrid 발급 | ✅ 정상 |
| 승인했는데 m2net 등록 실패 | ⚠️ 사주플랜 권한은 부여됨. csrid=NULL → 수동 재시도 |
| 반려 → 회원에게 알림톡 + 재신청 가능 | ✅ 정상 |
| 같은 회원이 동시에 2건 신청 pending | ❌ 데이터 사고. 이전 신청 처리 안 됨 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 신청 목록 (대기) | 사이드바 → 회원현황 → 상담사 신청 내역 |
| 신청자 상세 + 승인/반려 | 사이드바 → 회원현황 → 상담사 신청 내역 → 신청자 클릭 |
| 승인된 상담사 목록 | 사이드바 → 회원현황 → 상담사 리스트 |
