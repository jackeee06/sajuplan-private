# 상담사 상담 가능 상태 — 켜고 끄기 & 부재 처리

## 한 줄로 답하면
**상담사의 "상담 가능/부재" 상태는 상담사가 직접 켜고 끈 값이 그대로 유지됩니다. 앱을 꺼도, 장시간 접속을 안 해도 자동으로 오프라인이 되지 않습니다. (24시간 자동 부재중 기능은 ❌ 아직 없습니다 — 계획만 있던 것)**

## ⚠️ 2026-06-11 정정 — 예전 문서가 틀렸었음
이 문서는 한동안 "24시간 무응답이면 자동 부재중 전환된다"고 적혀 있었지만, **그 기능은 코드에 구현된 적이 없습니다.** (전용 cron 없음, `absent_status`·`last_active_at` 컬럼 DB에 없음.) 아래가 **실제 동작**입니다.

## 실제 동작 — 상태를 바꾸는 건 딱 4가지

```
① 상담사 본인 토글 (마이페이지)
   "지금 상담 가능 / 부재중" + 전화상담 ON·OFF + 채팅상담 ON·OFF
   → 켜둔 값 그대로 유지됨 (앱 꺼도 안 바뀜)

② 실제 상담 중 (m2net 자동)
   통화 시작 → CONN(통화중) / 채팅 시작 → CNCH(채팅중)
   상담 끝나면 → 원래 대기 상태로 자동 복귀

③ m2net 상태 푸시
   외부 m2net이 상태 변경을 알려오면 반영

④ 관리자 수동 변경
```

**앱 종료·로그아웃·장시간 미접속 → 상태 안 바뀜.** 로그인해도 "마지막 접속시각"만 갱신될 뿐 상담 상태는 손대지 않습니다.

## 그래서 생기는 일 (꼭 알아두세요)

상담사가 "상담 가능 + 채팅 ON"으로 **켜둔 채 앱을 끄면**, 회원 리스트엔 계속 **"채팅 대기"로 보입니다.** 회원이 채팅을 걸면:
- 상담사 휴대폰에 **푸시 알림**이 갑니다 → 앱을 켜서 응대해야 함
- 상담사가 응답을 안 하면 → **미응답(3분 자동취소)** 처리

→ 즉 **"대기로 보이는데 실제론 자리에 없는"** 상담사가 생길 수 있습니다. 이게 현재 구조의 한계입니다.

## 이런 상황을 보셨나요?

### "앱을 껐는데도 제가 상담가능으로 떠요"
→ **정상입니다.** 자동 오프라인 기능이 없어서, 직접 "부재중"으로 바꾸지 않는 한 켜둔 상태가 유지됩니다. 자리를 비울 땐 마이페이지에서 **"부재중"으로 직접 전환**하셔야 합니다.

### 회원: "대기라고 떠서 채팅 걸었는데 답이 없어요"
→ 상담사가 상태를 켜둔 채 앱을 꺼둔 경우일 수 있습니다. 3분 내 응답 없으면 자동 취소되고 차감도 없습니다. (1단계 대기에선 코인 차감 0)

### "휴가라 며칠 쉬고 싶어요"
→ 마이페이지에서 "부재중"으로 전환해 두시면 됩니다. 별도 휴가 모드는 없습니다.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 앱 껐는데 상담가능 유지됨 | ✅ 정상 (자동 오프라인 기능 없음) |
| 상담사가 토글로 부재중 전환 → 리스트에서 숨김 | ✅ 정상 |
| 통화/채팅 중 → CONN/CNCH, 끝나면 대기 복귀 | ✅ 정상 |
| 24시간 무응답인데 자동 부재 안 됨 | ✅ 정상 (그 기능 자체가 없음) |
| 토글 OFF 했는데 리스트에 계속 보임 | ❌ **사고**. m2net 동기화 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 상담사 현재 상태 (대기/상담중/부재) | 사이드바 → 회원현황 → 상담사 리스트 |
| 상태 수동 변경 | 상담사 리스트 → 상세 → 전화/채팅 사용 토글 |

## 향후 검토 (미구현)
- 앱 백그라운드/종료 감지 → 자동 부재
- 하트비트(주기 신호) 끊기면 N분 후 자동 부재
- 마지막 접속 N시간 경과 시 리스트에서 숨김 (가장 가벼운 안)
