# 추천인 시스템 v2 운영 가이드 (2026-06-10 실시간 적립 전환)

## 한 줄로 답하면
**기존 상담사(A)가 신규 상담사(B)를 추천하면, B가 상담을 할 때마다 그 상담 수익금의 일부(기본 1%)가 A의 수익금으로 실시간 적립됩니다. 추천 관계는 등록일 기준 3개월간 유효합니다. 관리자가 매월 지급 버튼을 누를 필요가 없습니다 — 자동입니다.**

---

## 🔴 2026-06-10에 무엇이 바뀌었나요?

**예전**: 매월 1일 정산할 때 한 달치 추천 수당을 한 번에 계산해서 지급했습니다. (관리자가 지급 버튼을 눌러야 했음)

**지금**: 피추천 상담사(B)가 **상담을 끝낼 때마다 그 자리에서 바로** 추천자(A)에게 추천수익금이 적립됩니다.
- 월별 배치(매월 1일 일괄 계산)는 폐지되었습니다.
- 관리자가 따로 지급 버튼을 누를 필요가 없습니다 — B가 상담하면 자동으로 A 수익금이 늘어납니다.
- B의 수익금에서 그만큼 빠져서 A에게 옮겨지는 구조라(제로섬), 회사가 추가로 부담하는 비용은 0원입니다.

> 용어: 상담사 추천 보상은 **"추천수익금"** 으로 부릅니다. ("추천 수당", "추천 포인트" 표현은 더 이상 쓰지 않습니다.)

---

## 왜 추천인 시스템을 만들었나요?

1. **상담사 네트워크 확장** — 광고 없이 기존 상담사의 입소문으로 신규 상담사를 유입할 수 있습니다. 기존 상담사가 검증한 신규 상담사는 품질이 더 높을 가능성이 큽니다.
2. **상담사 이탈 방지** — 추천 관계가 살아있는 동안 추천자(A)가 활동하는 신규 상담사를 챙길 유인이 생깁니다.
3. **초기 집중 지원** — 등록 후 3개월간 추천수익금을 지급해 신규 상담사 정착을 지원하고, 이후 자립을 유도합니다.

---

## 추천수익금 정책 (현재 설정값)

| 항목 | 값 | 설명 |
|------|------|---------|
| 요율 | B 상담 수익금의 **1%** | `promotion.referral_rate = 0.01` |
| 기간 | **3개월** | `promotion.referral_months = 3` · 등록일(상담사 승인일) 기준 |
| 3개월 초과 | **없음** | 관계 자동 만료 (`status = expired`) |
| 부담 주체 | **피추천자(B)** | B 수익금에서 차감 → A에게 이전 (제로섬, 회사 비용 0) |

> 예시: B가 상담 1건으로 수익금 10,000원을 벌면 → 그 즉시 A에게 100원(1%) 적립, B 수익금에서 100원 차감.
> 등록 당시의 요율·기간은 **스냅샷으로 고정**됩니다. 이후 정책을 바꿔도 기존 관계엔 영향이 없습니다.

---

## 추천 코드란?

- **형식**: 상담사 로그인 아이디(`mb_id`) 그대로 사용
- 별도 코드 발급 불필요 — 아이디가 곧 추천 코드
- **왜?** — 별도 코드 관리 시스템 없이 단순하게 운영하기 위해서입니다

신규 상담사 신청 시 "추천인 아이디" 입력란에 A의 mb_id를 입력하면 추천 관계 성립.

## 적립 흐름 (자동)

```
피추천 상담사(B)가 상담 종료
  ↓
B의 이번 상담 수익금(실수익) 확정
  ↓
B를 추천한 상담사(A)가 활성(active) 관계인지 확인
  ↓
관계 살아있으면 (3개월 이내):
  ↓
A 수익금(earning_balance) += 실수익의 1%   ← 즉시 적립
B 수익금(earning_balance) -= 동일 금액      ← 즉시 차감 (제로섬)
  ↓
다음 정산 때 A의 수익금에 자동 포함되어 입금
```

> ⚠️ 추천수익금은 **수익금(earning_balance)** 으로 적립됩니다.
> 상담 수익금과 똑같이 다음 정산에서 합산되어 입금됩니다.
> 관리자가 매월 따로 지급 버튼을 누르는 절차는 사라졌습니다.

## 경우의 수별 설명

### Q. "추천수익금은 언제 들어오나요?"
→ B가 상담을 끝내는 **바로 그 순간** A 수익금에 자동 적립됩니다. 월 1회 배치가 아닙니다.

### Q. "추천 관계가 몇 개월 됐는지 / 언제 만료되는지 확인하려면?"
→ 추천 관리 페이지 → `승인일`(registered_at) / `만료일`(expires_at) 컬럼 확인

### Q. "A가 탈퇴하거나 관계를 끊으면?"
→ 추천 관계 비활성(suspended/expired) 처리 → 이후 상담분부터 추천수익금 미적립

### Q. "B가 상담을 안 하면?"
→ 적립할 수익금이 없으므로 A에게도 0원 (B 활동 없으면 추천수익금 없음)

### Q. "추천 관계를 수동으로 등록하려면?"
→ 추천 관리 페이지 → 새 추천 관계 등록 버튼 → 추천자(A)·피추천자(B) 선택

### Q. "3개월이 지나 연장하고 싶으면?"
→ 현재 연장 기능 미구현. 3개월 후 자동 만료. (추후 검토 예정)

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 |
|------|------|
| B가 상담할 때마다 A에게 1% 적립 | ✅ 정상 |
| B 수익금에서 동일 금액 차감 | ✅ 정상 (제로섬) |
| 만료일 지난 관계인데 계속 적립 | ❌ 오류 — 즉시 확인 |
| B가 활동 안 해서 적립이 0원 | ✅ 정상 |
| 같은 상담 건이 두 번 적립됨 | ❌ 오류 — 멱등 처리되어 있으므로 발생 시 즉시 확인 |

## 운영자가 확인하는 위치

| 작업 | 메뉴 위치 |
|------|---------|
| 전체 추천 관계 목록 | 매출현황 → 상담사 추천 관리 |
| 추천 관계 수동 등록 | 추천 관리 → 새 추천 관계 등록 |
| 정책 요율·기간 변경 | 추천 관리 → 정책 설정 (슈퍼관리자 전용) |
| 적립 내역 확인 | 수익금 내역 (상담사 마이페이지) / 관리자 포인트 내역 |

> 참고: "이번 달 지급 버튼"은 실시간 적립 전환으로 더 이상 사용하지 않습니다.
