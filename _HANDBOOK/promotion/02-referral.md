# 추천인 시스템 v2 (2026-06-04 도입)

## 한 줄로 답하면

**기존 상담사가 신규 상담사를 추천하면, 신규 상담사가 벌어들이는 수익금의 일부(현재 1%)를 매월 자동으로 추천인에게 지급하는 시스템.**

---

## 전체 흐름 한눈에

```
① 기존 상담사 A → 본인 추천 코드(= 본인 아이디, 예: jackee) 확인 후 지인에게 공유
      ↓
② 신규 상담사 B → 상담사 신청서 작성 시 "추천인 코드" 입력
      ↓
③ 관리자가 신청 승인 → 사주플랜이 A↔B 추천 관계 자동 등록
      ↓
④ B가 매월 상담 → 수익금 발생
      ↓
⑤ 매월 1일 자정 정산 크론 실행
   - B의 정산 수익금에서 1% 자동 차감
   - A의 수익금에 "+추천 수당" 으로 자동 적립
      ↓
⑥ 사장님이 어드민에서 정산 확인 → 통장 송금 → [지급완료] 마킹
```

---

## 정책 상세

| 항목 | 현재 값 | 변경 방법 |
|---|---|---|
| 인센티브 요율 | **수익금의 1%** | 어드민 → 매출현황 → 추천 수당 → 슈퍼관리자 정책 패널 |
| 적용 기간 | **3개월** (승인일 기준) | 동일 |
| 기간 기산점 | 상담사 **승인된 날** | — |
| 재원 | B의 수익금에서 차감 | A안 (사장님 결정, 2026-06-04) |
| 지급 시점 | 매월 1일 정산 때 일괄 | settlement-cron 자동 |
| 원천징수 | 추천 수당도 수익금에 포함 → 정산 시 일괄 적용 | — |
| 회선비 | **미적용** (전화선과 무관) | — |
| 요율 변경 효력 | 변경 저장 즉시 → **신규 추천부터만 적용** | 기존 추천 관계는 등록 당시 요율 유지 |

---

## 추천 코드란?

- 각 상담사의 **로그인 아이디(mb_id)가 곧 추천 코드** (예: jackee)
- 상담사 승인 시 자동 설정. 별도 발급 불필요.
- 상담사 마이페이지 → 추천 현황에서 확인 + 카카오 공유 가능
- 대소문자 구분 없음 (`jackee` = `JACKEE`)
- 아이디를 바꾸면 추천 코드도 바뀜 (현재 아이디 변경 정책 없음)

---

## 추천 수당 계산 예시

| 시나리오 | 계산 |
|---|---|
| B의 이번 달 수익금 = 150,000원, 요율 1% | 추천 수당 = 1,500원 |
| B의 이번 달 수익금 = 9,200원 (23,000원 상담 1건, 40%) | 추천 수당 = 92원 |
| B의 이번 달 수익금 = 0원 (상담 없음) | 추천 수당 = 0원 (지급 없음) |

## 정산서에서 어떻게 보이나?

### 추천받은 상담사(B) 정산서

```
이달 수익금 합계     150,000원
추천인 수수료 차감    -1,500원   ← 1%, 찬물선생 추천 (3개월 중 2개월차)
───────────────────────────────
소계                 148,500원
원천징수 3.3%         -4,900원
회선비               -20,000원
실지급액             123,600원
```

### 추천한 상담사(A) 수익금 내역

```
[전화] 상담 수익      +9,200원
[추천 수당] B상담사    +92원    ← point_history rel_table='counselor_referral'
합계                  +9,292원
```

## 운영자가 자주 받는 질문

### "추천 코드 입력했는데 수당이 안 들어와요"

→ 확인 순서:
1. 어드민 → 매출현황 → 추천 수당 → 해당 추천 관계 **status 확인** (active/expired/suspended)
2. `expires_at` 이 지났으면 → 3개월 만료됨 (정상)
3. 해당 월에 B의 정산이 0원이면 → 수당도 0 (정상)
4. `counselor_referral_payment` 테이블에서 해당 월 지급 기록 확인

### "추천 코드를 어디서 확인하나요?"

→ 상담사 마이페이지(앱) → 추천 현황 탭에서 본인 코드 확인 + 카카오 공유 가능

### "기간이 지나서 수당이 끊겼는데 연장하고 싶어요"

→ 현재 연장 기능 없음. 필요 시 어드민에서 수동 새 추천 관계 등록 (expires_at 새로 설정 가능).

### "요율을 바꾸면 기존 추천인들도 바뀌나요?"

→ **바뀌지 않습니다.** 등록 당시 요율(rate_snapshot)이 해당 추천 관계에 고정 저장됩니다.

### "자격정지된 상담사가 추천받은 건은 어떻게 되나요?"

→ status='suspended' 로 변경하면 다음 정산부터 해당 추천 수당 지급 중단.

## 운영자 확인 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 추천 관계 전체 목록 | 어드민 → 매출현황 → 추천 수당 |
| 정책 변경 (요율·기간) | 동일 페이지 상단 슈퍼관리자 전용 패널 |
| 이번 달 지급 대상 | 동일 페이지 → status=active + 지급 버튼 |
| 상담사 추천 코드 확인 | 어드민 → 회원 → 상담사 상세 → referral_code 컬럼 |
| 수당 이력 (DB) | point_history WHERE rel_table='counselor_referral' |

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 |
|---|---|
| B의 수익금 있는데 A에게 수당 없음 | ⚠️ counselor_referral_payment 확인. 기간 만료 여부 체크 |
| 수당이 0원 | ✅ B의 이번달 수익금이 0원이면 정상 |
| expires_at 지남 → status=expired | ✅ 정상 만료 |
| 같은 referral_id + month 에 이중 지급 | ❌ 버그 — UNIQUE constraint 위반이라 실제 불가 |
| rate_snapshot 이 현재 요율과 다름 | ✅ 정상 — 등록 당시 요율 스냅샷 |
