# 추천인 시스템 v2 운영 가이드 (2026-06-04 도입, 2026-06-04 최종 정리)

## 한 줄로 답하면

**기존 상담사가 신규 상담사를 추천하면, 신규 상담사가 벌어들이는 수익금의 1%를 매월 자동으로 추천인에게 지급하는 시스템.**

---

## 전체 흐름 한눈에

```
① 기존 상담사 A → 앱 추천 현황에서 본인 추천 코드 확인 (= 본인 아이디)
      ↓
② 카카오로 공유하거나 코드를 직접 알려줌
      ↓
③ 신규 상담사 B → 상담사 신청서 작성 시 "추천인 코드" 란에 A의 아이디 입력
   (입력 즉시 "○○선생 맞습니다" 닉네임 확인)
      ↓
④ 관리자가 신청 승인 → 시스템이 A↔B 추천 관계 자동 등록
      ↓
⑤ B가 매월 상담 → 수익금 발생
      ↓
⑥ 매월 1일 자정 자동 정산
   - B 수익금에서 1% 차감
   - A 수익금에 +1% 적립
      ↓
⑦ 사장님이 어드민에서 정산 확인 → 통장 송금 → [지급완료] 마킹
```

---

## 정책 상세

| 항목 | 현재 값 | 비고 |
|---|---|---|
| 인센티브 요율 | **수익금의 1%** | 슈퍼관리자 전용 변경 가능 |
| 적용 기간 | **3개월** | 승인일 기준 |
| 기간 기산점 | 상담사 **승인된 날** | 신청일 아님 |
| 재원 | **B(피추천인)의 수익금에서 차감** | 회사 부담 아님 |
| 지급 시점 | 매월 1일 자동 | settlement-cron |
| 원천징수 | 추천 수당도 수익금에 포함 → 정산 시 일괄 적용 | 별도 세금 없음 |
| 회선비 | **미적용** | 전화선 비용과 무관 |
| 요율 변경 효력 | **신규 추천부터만 적용** | 기존 추천 관계는 등록 당시 요율 고정 |

> ⚠️ **요율/기간은 슈퍼관리자만 변경 가능**: 어드민 → 매출현황 → 추천 수당 → 상단 정책 패널

---

## 추천 코드란?

- **본인 로그인 아이디(mb_id)가 곧 추천 코드** (예: 아이디가 `jackee`면 추천 코드도 `jackee`)
- 상담사 승인 시 자동 설정 — 별도 발급 절차 없음
- **대소문자 구분 없음** (`jackee` = `JACKEE` = `Jackee` 모두 동일)
- 아이디 변경 시 추천 코드도 변경됨 (현재 아이디 변경 기능 없음)
- 확인 위치: 앱 → 상담사 모드 → 마이페이지 → 추천 현황

## 앱 화면별 표시 방식 (2026-06-04 기준)

### 📱 추천 현황 페이지 (앱 → 상담사 모드 → 마이 → 추천 현황)

두 섹션으로 구성:

**① 내가 추천한 상담사 (추천인 A 시점)**
```
내 추천 코드: jackee  [복사]  [카카오로 공유하기]

누적 추천 수당: 4원     추천한 상담사: 1명

추천한 상담사
  찬물선생   승인 2026-06-01 · 만료 2026-09-01 · 1% · 3개월
             진행 중   +4원
```

**② 나를 추천한 상담사 (피추천인 B 시점) — 추천받은 경우만 표시**
```
나를 추천한 상담사
  라온선생   1개월/3개월 진행 중 · 수익금의 1% 차감
  만료: 2026-09-01         진행 중   -4원 차감됨
```

> 추천받은 적 없으면 "나를 추천한 상담사" 섹션 자체가 안 보임

---

### 📱 수익금 내역 (앱 → 상담사 모드 → 수익금 → 수익금 내역 탭)

추천 수당 항목이 일반 상담 내역과 함께 리스트에 표시됨:

```
날짜          내역                고객명   구분   수익금
2026-06-02    [전화]수익금         타*      선불   +9,200원
2026-06-01    [추천 수당] jackee   -        -      +4원      ← 추천인에게 초록색 표시
2026-05-23    [채팅]수익금         찬*생    선불   +400원
```

피추천인 B(찬물선생)에게는:
```
2026-06-01    [추천 수당 차감]     -        -      -4원      ← 빨간색 표시
```

---

### 📱 실시간 정산 탭 상단 카드

```
이번달 누적 수익금
전달 400원   이달 9,200원
추천 수당 차감  -4원            ← 피추천인에게만 표시
실수령 예상  8,085원
정산비 9,196 − 공제 1,111(부가세+원천징수) = 8,085원   ← 계산 근거
```

추천인 A(라온선생)에게는:
```
추천 수당 +4원                  ← 초록색으로 표시
```

---

### 📱 실시간 정산 탭 breakdown 카드

```
쿠폰상담    충전+후불 상담    기타정산비    정산비전체
  0원          9,200원          -4원         9,196원
```
- `기타정산비 -4원` = 추천 수당 차감 (B의 경우)
- `기타정산비 +4원` = 추천 수당 적립 (A의 경우, 상담 수익 없을 때)

## 추천 수당 계산 예시

| 시나리오 | 계산 |
|---|---|
| B의 이번 달 수익금 = 150,000원, 요율 1% | 추천 수당 = 1,500원 차감/적립 |
| B의 이번 달 수익금 = 9,200원 | 추천 수당 = 92원 (7월 1일 크론 처리) |
| B의 이번 달 수익금 = 0원 (상담 없음) | 수당 = 0원, 기록 없음 |
| B의 이번 달 수익금 = 4원 (추천 수당만 있을 때) | 실수령 = 3원 (부가세 1원 공제) |

## 누적 추천 수당이란?

- **지금까지 정산으로 실제 지급된 금액의 합계**
- 아직 정산 안 된 잔액이 아님
- 7월 1일 크론이 돌기 전까지는 6월 수당은 "누적"에 포함되지 않음

## 운영자가 자주 받는 질문

### "추천 코드 입력했는데 수당이 안 들어와요"
확인 순서:
1. 어드민 → 매출현황 → 추천 수당 → 해당 추천 관계 **status 확인** (active/expired/suspended)
2. `expires_at` 지남 → 3개월 만료됨 (정상)
3. 해당 월 B의 수익금이 0원이면 → 수당도 0 (정상)
4. 아직 이번 달 정산 크론 안 돌았으면 → 다음 달 1일 대기 (정상)

### "추천 코드를 어디서 확인하나요?"
앱 → 상담사 모드 → 마이페이지 → 추천 현황 → 내 추천 코드 표시

### "추천 코드가 뭔지 모르겠어요"
본인 로그인 아이디입니다. 예: 아이디가 `raonteacher`이면 추천 코드도 `raonteacher`.

### "기간이 지나서 수당이 끊겼어요, 연장하고 싶어요"
현재 연장 기능 없음. 어드민 → 추천 수당 → 수동으로 새 추천 관계 등록 가능 (expires_at 새로 설정).

### "요율을 바꾸면 기존 추천인들도 바뀌나요?"
**바뀌지 않습니다.** 등록 당시 요율(rate_snapshot)이 해당 추천 관계에 고정. 신규 승인부터만 새 요율 적용.

### "자격정지된 상담사가 추천받은 건은?"
어드민에서 해당 추천 관계 status='suspended' 변경 → 다음 정산부터 차감/적립 중단.

### "내 수익금에서 왜 -4원이 빠졌어요?"
추천 수당 차감입니다. 추천 현황 탭에서 "나를 추천한 상담사" 섹션에서 누가 추천했는지 확인 가능.

### "추천 수당이 얼마나 됐는지 어디서 봐요?"
- 앱 추천 현황 → 누적 추천 수당 금액
- 수익금 내역 탭 → `[추천 수당]` 항목으로 월별 확인

### "신청서에 추천 코드 입력했는데 닉네임이 안 뜨면?"
존재하지 않는 아이디입니다. 정확한 아이디를 다시 확인 후 입력.

| 추천 관계 비활성화 | 어드민 → 추천 수당 → 해당 행 → 비활성 버튼 |
| 이번 달 수당 지급 기록 | 어드민 → 추천 수당 → 필터: 이번 달 |

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 | 대응 |
|---|---|---|
| B 수익금 있는데 A에게 수당 없음 | ⚠️ 확인 필요 | 추천 관계 status, 기간 만료 체크 |
| 수당이 0원 | ✅ B의 이달 수익금이 0원이면 정상 | — |
| expires_at 지남 → status=expired | ✅ 정상 만료 | — |
| 이중 지급 발생 | ❌ 실제 불가 (UNIQUE 제약) | DB 직접 확인 |
| rate_snapshot이 현재 요율과 다름 | ✅ 등록 당시 요율 스냅샷 | 정상 |
| 추천 현황에 "나를 추천한 상담사" 안 보임 | ✅ 추천받은 적 없으면 정상 미표시 | — |
| 수익금 내역에 추천 수당 항목 없음 | ✅ 이번 달 B 수익 없거나 크론 아직 미실행 | 다음 달 1일 후 확인 |
