# 회원·상담사·관리자 역할은 어떻게 나뉘나요?

## 한 줄로 답하면
**한 사람이 여러 역할 동시 보유 가능. 회원 + 상담사 듀얼이 흔하고, 관리자는 별도. 슈퍼관리자는 사장님 본인용.**

## 4가지 역할

| 역할 | 권한 |
|---|---|
| **회원** (member) | 채팅·통화 신청, 코인 충전, 후기 작성 |
| **상담사** (counselor) | 채팅·통화 수신, 정산·선지급 신청 + 회원 역할 |
| **관리자** (admin, 일반) | 관리자 페이지 접근, 환불·정산 처리, 회원 관리 |
| **슈퍼관리자** (super) | 위 + 영업이익 시뮬레이터, 슈퍼 승격 권한 |

## 듀얼 계정 (회원 ↔ 상담사)

한 사람이 회원 + 상담사 양쪽 활동 가능:
- 사이드바 회원 영역: 마이페이지, 채팅 신청
- 사이드바 상담사 영역: incoming, 정산
- 상단 핑크/보라 바 (ModeIndicator) 가 현재 모드 표시
- URL 경로로 자동 모드 (사주플랜이 알아서)

→ 자세한 건 [회원↔상담사 모드 전환](member/04-dual-account)

## 슈퍼관리자 vs 일반관리자

### 슈퍼 전용 (2가지만)
1. **슈퍼 승격 권한** — 다른 관리자를 슈퍼로 만들기
2. **비밀 수치 (영업이익)** — 영업이익률·회사 마진·비밀 단가

### 일반관리자 전권
- 회원 관리 (정지, 정보 수정)
- 결제, 환불, 정산 처리
- 콘텐츠, 알림, 게시판
- 일상 운영 모든 영역

## 이런 상황을 보셨나요?

### "이 회원이 상담사도 하고 싶대요" 라고 문의

→ 회원 마이페이지 → 상담사 신청 페이지 → 정보 입력 → 운영자 승인. 승인되면 회원 역할은 유지 + 상담사 추가 (듀얼).

### "일반관리자가 영업이익 시뮬레이터 접근 시도해요"

→ ✅ 차단됨 (정상). 슈퍼 전용 영역. 일반관리자에게는 메뉴 자체 안 보임.

### "관리자 한 명 더 추가하고 싶어요"

→ 슈퍼관리자만 가능. 사이드바 → 권한관리 → 관리자 계정 → 등록. 일반관리자로 등록.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 회원이 상담사 듀얼로 등록됨 | ✅ 정상 |
| 일반관리자가 환불 처리 | ✅ 정상 (일반 전권) |
| 일반관리자가 영업이익 시뮬레이터 접근 | ❌ 차단 (정상) |
| 상담사 본인이 자기 등급 직접 변경 시도 | ❌ 차단 (운영자 전용) |
| 회원이 다른 회원의 정보 조회 시도 | ❌ 차단 (정상) |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 전체 회원 목록 | 사이드바 → 회원현황 → 고객 리스트 |
| 상담사 목록 | 사이드바 → 회원현황 → 상담사 리스트 |
| 듀얼 계정 (회원+상담사) | 상담사 목록에서 role='counselor' 인 회원 모두 |
| 관리자 계정 | 사이드바 → 권한관리 → 관리자 계정 (슈퍼만 접근) |
