# 회원 가입 흐름

## 한 줄로 답하면
**phone 인증 → ID/비밀번호 입력 → m2net 자동 등록 → 가입 완료. m2net 등록 실패 시 사주플랜 가입도 자동 취소 (회원 안전 우선).**

## 가입 단계

```
phone 입력 → SMS 인증번호 (5분 유효) → 확인
  ↓
ID, 비밀번호 (8자+ 영문/숫자), 닉네임 입력
  ↓
사주플랜이 회원 등록 + m2net 자동 등록
  ↓
m2net 등록 성공 → 가입 완료 + 자동 로그인
m2net 등록 실패 → 사주플랜 가입도 취소 (롤백)
```

## 회원가입 코인 (가입 보상)

- 가입 완료 시 **회원가입 코인 10,000 즉시 지급** (`register_point`, 관리자 설정값 기준).
- **로컬(아이디 직접) 가입 · 소셜(카카오/네이버) 가입 모두 동일 지급.** (2026-06-10 수정 전엔 소셜만 누락됐었음 → 9명 소급 지급 완료)
- **회원가입 쿠폰은 폐지됨** (2026-06-07). 쿠폰 대신 즉시 코인으로 통일.

## 추천인 코드 입력 (옵션)

신규 회원이 추천인 코드 입력 시:
- 양쪽 (신규 회원 + 추천인) 에 무료 코인 보상
- 자세한 건 [추천인 시스템](promotion/02-referral)

## 이런 상황을 보셨나요?

### 회원이 "가입했는데 통화 안 돼요" 라고 클레임

→ m2net 등록 실패 가능성. 사이드바 → 회원현황 → 고객 리스트 → 상세 에서 m2net 등록 상태 확인.

### "같은 폰번호로 다른 ID 만들 수 있나요?"

→ 정책: 한 사람 한 ID. phone 중복 가능하지만 보통 같은 사람으로 봄.

### "추천인 코드 입력했는데 보상 안 들어왔어요"

→ 가입 직후 즉시 양쪽 적립. 안 들어왔으면 코드 잘못 입력 가능. 코드 확인 후 운영자 수동 정정 가능.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 가입 후 m2net 등록 자동 완료 | ✅ 정상 |
| 로컬·소셜 가입 모두 회원가입 코인 10,000 지급 | ✅ 정상 (2026-06-10~) |
| 2026-06-10 이전 소셜 가입자 코인 0 | ⚠️ 과거 버그. 9명 소급 완료 |
| m2net 등록 실패 → 사주플랜 가입도 자동 취소 | ✅ 의도 (회원 안전) |
| 가입 후 m2net 등록 빠짐 | ❌ **사고**. 즉시 점검 |
| 추천인 코드 입력 → 즉시 양쪽 보상 적립 | ✅ 정상 |
| 같은 phone 으로 2번째 ID 가입 시도 | ⚠️ 가능하나 비추천 (한 사람 한 ID 원칙) |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 회원 가입 이력 | 사이드바 → 회원현황 → 고객 리스트 |
| m2net 등록 상태 | 사이드바 → 회원현황 → 고객 리스트 → 상세 |
| 추천인 통계 | 사이드바 → 매출현황 → 추천수익금 |
