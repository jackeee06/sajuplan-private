# 회원이면서 상담사인 경우 화면이 어떻게 동작해요?

## 한 줄로 답하면
**한 사람이 회원 + 상담사 두 역할 가질 때, 사주플랜이 URL 경로로 자동 모드 구분. 상단 핑크/보라 바가 현재 모드 표시.**

## 모드 자동 구분

| URL 영역 | 모드 | 상단 바 색 |
|---|---|---|
| 홈, 마이페이지, 채팅 신청 | 회원 모드 | 핑크 |
| 상담사 마이페이지, 정산, incoming | 상담사 모드 | 보라 |

→ 사용자가 메뉴 클릭 시 사주플랜이 알아서 모드 전환.

## 모드 인디케이터 (상단 핑크/보라 바)

듀얼 계정 사용자에게만 표시:
- 항상 노출 (X 클릭으로 숨김 가능)
- 모드 전환 시 자동 토스트 알림 ("회원 모드로 전환됨" 등)
- 큰 안내 배너 (예: 상담사 모드에서 홈 진입 → "이 화면은 회원 모드에서 사용 가능" 안내)

## 이런 상황을 보셨나요?

### "회원이 상담사 모드에서 충전하려는데 안 보여요" 라고 클레임

→ 정상. 충전 = 회원 영역. 상담사 모드일 때는 다른 메뉴 (정산, incoming 등) 만 보임. 회원 모드로 전환 후 충전 가능.

### "상단 핑크 바 거추장스러워요"

→ X 클릭으로 숨김 가능. 좌상단 작은 닷 클릭하면 다시 노출.

### "일반 회원 (상담사 X) 인데 모드 바가 보여요"

→ ❌ 사고. 일반 회원에게는 모드 개념 없음. `!isCounselor` 가드 확인.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 듀얼 사용자가 홈 진입 → 핑크 바 (회원 모드) | ✅ 정상 |
| 듀얼 사용자가 상담사 마이페이지 → 보라 바 (상담사 모드) | ✅ 정상 |
| X 클릭 후 영구 숨김 | ✅ 정상 (localStorage 저장) |
| 모드 전환 시 자동 토스트 | ✅ 정상 |
| 일반 회원에게 모드 바 노출 | ❌ 가드 누락. 점검 |
| 채팅방 안에서 모드 바 보임 | ❌ 채팅방은 자동 숨김. 점검 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 듀얼 계정 회원 목록 | 사이드바 → 회원현황 → 상담사 리스트 (role='counselor' 인 회원 모두) |
| 모드별 사용 패턴 | 통계 페이지 (개발 가능) |
