# 출석 보상은 어떻게 동작하나요?

## 한 줄로 답하면
**회원이 매일 1회 출석 클릭하면 무료 코인 적립. 하루 1회만 가능. 운영자가 보상 금액 / 연속 보너스 설정 가능.**

## 동작

```
회원이 사주플랜 진입 → 출석 버튼 클릭
  ↓
사주플랜이 오늘 이미 출석했는지 확인
  ↓
첫 출석이면 → free_balance 증가 + 출석 알림 (토스트)
이미 출석했으면 → "이미 출석" 안내
```

## 이런 상황을 보셨나요?

### 회원이 "출석했는데 코인 안 들어왔어요" 라고 클레임

→ 가능 원인:
1. 이미 오늘 출석함 (재확인)
2. 사주플랜 일시 사고

→ 답변: "마이페이지 → 포인트 (코인 내역) 에서 오늘 출석 적립 확인 부탁드립니다."

### "자정 직후 (00:01) 출석했는데 어제로 인정됐어요"

→ 사주플랜은 KST 자정 기준. 00:01 = 새 날. 만약 어제로 인정됐다면 사고.

### "연속 출석 7일 했는데 보너스 없어요"

→ 연속 출석 보너스는 운영자 설정 사항. 사이드바 → 회원현황 → 출석 관리 에서 정책 확인.

## 회원이 자주 묻는 질문

### Q. "왜 매일 출석해도 보상이 작아요?"

운영자가 설정한 보상 금액. 정책 변경 가능. 시즌별 행사 시 보너스 가능.

### Q. "출석 안 한 날 복원 가능해요?"

X. 출석 = 능동적 클릭. 사주플랜이 자동 출석 처리 안 함.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 매일 1회 출석 → free_balance 증가 | ✅ 정상 |
| 같은 날 2번 시도 → 거부 | ✅ 정상 |
| 자정 (KST) 기준 새 날 출석 가능 | ✅ 정상 |
| 출석 보상이 paid_balance 로 적립 | ❌ 정책 위반 (free 로 가야 함) |
| 7일 연속 후 보너스 없음 | 정책 결정 사항. 운영자가 설정 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 출석 통계 | 사이드바 → 회원현황 → 출석 관리 |
| 일별 보상 금액 설정 | 사이드바 → 회원현황 → 출석 관리 → 설정 |
| 회원별 출석 이력 | 사이드바 → 회원현황 → 고객 리스트 → 상세 → 출석 탭 |
