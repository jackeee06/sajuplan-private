# 단골 / 즐겨찾기 (favorites)

## 한 줄로 답하면
**회원이 마음에 드는 상담사의 하트를 누르면 단골 등록. 단골 페이지에서 빠르게 다시 찾을 수 있고, 단골 상담사가 접속하면 홈 배너로 안내한다. 같은 상담사를 두 번 눌러도(멱등) 안전하며, 비로그인이면 로그인 안내가 뜬다. 본인은 자기 자신을 단골 등록할 수 없다.**

> 사용자 흐름(추가·조회·제거·본인 차단) 중심 안내는 [user/04-favorites](../user/04-favorites) 와 동일 주제다. 운영자 확인 위치·DB 레퍼런스는 이 문서를 본다.

## 동작

```
회원이 상담사 카드(또는 상세) → 하트 아이콘 클릭
  ↓
사주플랜에 단골 등록 (하트가 핑크색으로)
  ↓
회원 마이페이지 → 단골 페이지에 등록 최신순으로 표시
  ↓
단골 상담사 접속 시 → 홈 진입 시 "단골 N명 접속중" 배너 (옵션)
```

## 이런 상황을 보셨나요?

### 회원이 "단골 등록 안 돼요" 라고 클레임

→ 이미 등록되어 있을 가능성. 하트가 핑크색이면 이미 단골이다. 흰색이면 미등록.
→ 비로그인 상태면 하트를 눌러도 로그인 안내 모달이 먼저 뜬다 (로그인 후 다시 눌러야 등록).

### 회원이 "본인을 단골 등록하려는데 안 돼요"

→ **정상 차단**이다. 회원⇄상담사 듀얼계정 사용자가 자기 자신을 단골 등록하려 하면 "본인을 단골 등록할 수 없습니다" 가 뜬다 (2026-06-11 차단 강화).

### "단골 해제했는데 단골 페이지에 남아 있어요"

→ 캐시일 가능성. 새로고침하면 사라진다. 1시간 이상 지속되면 점검.

## 회원이 자주 묻는 질문

### Q. "단골 등록 한도 있나요?"

정책상 무제한. 다만 너무 많으면 단골 페이지 정렬이 번잡할 수 있다.

### Q. "같은 상담사를 두 번 눌렀어요. 중복 등록되나요?"

아니다. 같은 상담사는 한 번만 등록된다(멱등). 더블클릭·재시도 해도 안전.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 하트 클릭 → 단골 등록 + 핑크색 변경 | ✅ 정상 |
| 하트 다시 클릭 → 단골 해제 | ✅ 정상 |
| 이미 단골인 상담사 재등록 시도 | ✅ 정상 (멱등 — 중복 안 생김) |
| 비로그인 하트 클릭 → 로그인 안내 모달 | ✅ 정상 |
| 본인이 본인을 단골 등록 시도 | ✅ 정상 차단 (403) |
| 단골 상담사 접속 → 홈 "단골 N명 접속중" 배너 | ✅ 정상 |
| 단골 해제했는데 페이지에 남음 | ⚠️ 캐시. 새로고침 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 회원별 단골 상담사 목록 | 사이드바 → 회원현황 → 고객 리스트 → 상세 |
| 상담사별 팬 수(fan_count) | 상담사 상세 (단골 등록 수 실시간 집계) |
| 단골 통계 (인기 상담사) | 운영 SQL (.tech 참조) |
