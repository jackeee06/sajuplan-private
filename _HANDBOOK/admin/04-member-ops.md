# 회원·상담사 관리 (정지·강제로그아웃·모니터링)

## 한 줄로 답하면
**관리자 페이지에서 회원/상담사 검색·조회·정지·모니터링 가능. 데이터 영구 삭제는 절대 X (soft delete).**

## 주요 운영 작업

### 회원 관리
- 회원 검색 (mb_id, phone, 닉네임)
- 잔액 / 결제·사용 이력 조회
- 정지 / 복원 토글
- 강제 로그아웃
- 운영자 메모

### 상담사 관리
- 상담사 목록 + 상세
- 등급 변경
- 부재중 토글 (수동)
- 상담 이력 / 정산 / 선지급

### 환불·수동 정정
- 회원 클레임 → 수동 환불
- 차감 오류 → 포인트 수동 정정
- 정산 누락 → 수동 추가

## 이런 상황을 보셨나요?

### "어뷰징 의심 회원 발견했어요"

→ 정지 처리. 사이드바 → 회원현황 → 고객 리스트 → 회원 검색 → 상세 → 상태 "정지". 사유 메모 필수.

### "회원이 다른 사람 명의로 가입한 것 같아요"

→ 같은 phone 다수 ID 체크. 같은 카드 다수 결제 체크. 의심 시 정지.

### "회원 정보 수정해도 되나요?"

→ 정책 결정. 일반적으로 운영자 수정 X (회원 본인이). 닉네임 / 프로필만 운영자 수동 변경 가능 (사유 필요).

### "정지된 회원의 옛 채팅·결제 이력은?"

→ 보존됨. soft delete 정책. DB 직접 삭제 절대 금지.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 회원 정지 → 로그인 차단 | ✅ 정상 |
| 정지된 회원의 옛 이력 보존 | ✅ 정상 (soft delete) |
| 정지된 회원이 다른 phone 으로 재가입 | ⚠️ 어뷰징 의심. 검토 |
| DB 에서 회원 직접 삭제 | ❌ **절대 금지** (FK 제약 + 이력 손실) |
| 강제 로그아웃 → 즉시 세션 종료 | ✅ 정상 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 회원 검색 / 상세 | 사이드바 → 회원현황 → 고객 리스트 |
| 상담사 검색 / 상세 | 사이드바 → 회원현황 → 상담사 리스트 |
| 환불 처리 | 사이드바 → 매출현황 → 환불 이력 |
| 포인트 수동 정정 | 사이드바 → 매출현황 → 포인트 관리 |
| 채팅 이력 (사고 추적) | 사이드바 → 상담관리 → 채팅내역 리스트 |
