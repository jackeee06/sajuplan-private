# 코인 환불은 어떨 때 해주나요?

## 한 줄로 답하면
**채팅 5초 이내 자동 환불 (G 정책) + 통화 50초 이내 자동 환불 (고객보호비용) + 운영자 수동 환불 3가지가 있어요.**

## 환불 종류

### 1. 자동 환불 — 채팅 5초 이내 (G 정책)
- 회원이 상담사 입장 직후 5초 안에 종료 → 차감 코인 전액 자동 복원
- 한도: 하루 2회, 일주일 4회 (어뷰징 차단)
- 사주플랜이 자동 처리. 운영자 개입 X.

### 2. 자동 환불 — 통화 50초 이내 (고객보호비용)
- 회원이 통화 시작 후 50초 안에 종료
- 사용 시간 미달 → 회원에게 차감 코인 환불
- 상담사에게는 정산 미발생 (사용 시간 0)
- 회사가 m2net 통화료만 비용 흡수
- 사주플랜이 자동 처리.

### 3. 수동 환불 — 운영자가 처리
- 회원 클레임 / 사고 발생 / 어뷰징 의심 등
- 사이드바 → 매출현황 → 환불 이력 페이지에서 처리
- 사유 입력 필수 (감사 추적)

## 이런 상황을 보셨나요?

### 회원이 "상담사 인사하고 바로 끊었는데 코인 빠졌어요" 라고 클레임

→ 5초 안에 끝났으면 자동 환불됨. 환불 안 됐다면:
- 가능 원인 1: 5초 초과
- 가능 원인 2: G 정책 한도 도달 (일 2회 / 주 4회)
- 가능 원인 3: 시스템 사고

→ 답변: "채팅 시간 확인하고 한도 초과 안 했으면 수동 환불 처리하겠습니다."

### 회원이 "통화 30초만 했는데 코인 다 빠졌어요"

→ 50초 이내면 자동 환불되어야 함. 안 됐다면 점검 필요.

### 회원이 "60분 채팅 신청했는데 30분만 쓰고 끝났어요. 30분 환불해주세요"

→ ❌ **환불 불가**. 사주플랜은 선택한 시간 전체 차감 정책. 사용 시간 미만 환불 X. (선결제 정책 — 자세한 건 [채팅 코인 차감 정책](chat/01-prepaid-policy))

### 회원이 "충전 잘못했어요. 환불해주세요"

→ ✅ 미사용 잔액 환불 가능. 사용한 코인은 환불 X. 사이드바 → 매출현황 → 환불 이력 에서 수동 처리.

## 회원이 자주 묻는 질문

### Q. "왜 자동 환불 한도가 있어요?"

한도 없으면 같은 회원이 매일 5초 환불 받는 어뷰징 가능. 정상 사용자 보호 + 어뷰징 차단 균형.

→ 답변: "5초 환불은 실수 보호용이라 한도가 있습니다 (일 2회 / 주 4회). 한도 외엔 사용한 것으로 간주됩니다."

### Q. "수동 환불은 얼마 만에 처리돼요?"

운영자가 검토 후 즉시 처리 가능. 사장님 영업시간 기준 보통 1-2시간 내.

### Q. "환불 받은 코인은 다시 돌려받은 거랑 같나요?"

네. paid_balance (충전 코인) 에 복원. 다시 사용 가능.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 5초 이내 채팅 종료 → 자동 전액 환불 | ✅ 정상 (G 정책) |
| 5초 + 1초 = 6초 종료 → 환불 X | ✅ 정상 (한도 초과) |
| 같은 회원 일 3번 5초 환불 → 3번째 거부 | ✅ 정상 (일 2회 한도) |
| 통화 40초 → 환불 처리 | ✅ 정상 (고객보호비용) |
| 환불 처리됐는데 잔액에 안 들어옴 | ❌ **사고**. 즉시 점검 |
| 운영자가 사유 입력 없이 환불 | ⚠️ 감사 추적 어려움. 사유 필수 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 환불 이력 (자동 + 수동) | 사이드바 → 매출현황 → 환불 이력 |
| 고객보호비용 (50초 이내 통화 환불) | 사이드바 → 매출현황 → 고객보호비용 내역 |
| G 정책 5초 환불 받은 회원 | 사이드바 → 매출현황 → 환불 이력 → 사유 컬럼 "G 환불" |
| 수동 환불 처리 | 사이드바 → 매출현황 → 환불 이력 → 환불 등록 |
