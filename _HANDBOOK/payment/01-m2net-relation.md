# 사주플랜과 m2net의 관계는 어떻게 되나요?

## 한 줄로 답하면
**m2net (외부 통신 회사) 이 통화/채팅의 전화선 역할을 합니다. 사주플랜은 m2net 에게 회원·상담사 정보 + 결제 잔액을 동기화 시켜요.**

## m2net 이 하는 일

| 역할 | 내용 |
|---|---|
| **가상번호 발급** | 통화 시 070-XXXX 가상번호로 회원·상담사 연결 (실제 번호 공개 X) |
| **통화 라우팅** | 회원이 070 가상번호 다이얼 → m2net 이 상담사 폰으로 연결 |
| **채팅 wss** | 채팅 메시지 양방향 통신 |
| **잔액 보유** | m2net 측 자체 회원 잔액 → 단가 기준 차감 |

## 이런 상황을 보셨나요?

### 회원이 "전화 안 걸려요" 라고 클레임

→ 가능 원인 (운영자가 확인할 것):
1. 회원/상담사 폰이 070 번호 차단 설정 → 회원에게 차단 해제 안내
2. 상담사 부재중 (앱 미진입) → 회원에게 다른 상담사 안내
3. m2net 측 서비스 사고 → 사장님에게 보고 + m2net 측 확인

### "회원 잔액이 사주플랜과 m2net 측에서 다르대요"

→ 두 시스템 잔액이 일시적 어긋날 수 있음. 사주플랜은 채팅 끝날 때마다 m2net 측에 강제 동기화. 큰 어긋남이면 자동 정정 시스템이 처리.

→ 답변: "잠시 후 자동 동기화됩니다. 1시간 지나도 어긋나면 사장님께 알려주세요."

### "충전 했는데 m2net 측에는 반영 안 됐대요"

→ 충전 즉시 사주플랜 + m2net 양쪽 적립 시도. m2net 측 누락 시 자동 정정 시스템이 30초 안에 동기화.

## 사주플랜 ↔ m2net 동기화 시점

| 사건 | 동기화 |
|---|---|
| 회원 가입 | 사주플랜 → m2net (회원 등록, 가상번호 발급용 ID 발급) |
| 상담사 신청 승인 | 사주플랜 → m2net (상담사 등록) |
| 충전 (결제) | 사주플랜 + m2net 양쪽 적립 |
| 채팅·통화 종료 | 사주플랜 잔액 → m2net 측 강제 일치 |

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 가입 후 m2net 측 등록 OK | ✅ 정상 |
| 가입 → m2net 등록 실패 → 사주플랜 가입도 취소 | ✅ 의도 (회원 안전) |
| 상담사 신청 승인 후 m2net 등록 실패 | ⚠️ 사주플랜 상담사는 유지. 운영자 수동 재시도 |
| 잔액 일시적 어긋남 (몇 분 내) | ✅ 자동 정정 시스템 진행 중 |
| 잔액 어긋남 1시간+ 유지 | ❌ **사고**. 보고 + 점검 |
| 통화 시도 → 가상번호 발급 안 됨 | ❌ **사고**. m2net 측 응답 점검 |
| **선결제 채팅인데 상담내역 금액(amt)이 0** | ✅ **정상.** 선결제는 채팅 시작 때 한 번에 받고, 끝날 땐 추가로 안 받아요. 0이 맞습니다 |
| 회원 보유 코인이 "코인 내역" 합보다 많아 보임 | ✅ 정상. 충전은 별도 결제장부에 있고 내역엔 일부만 표시돼요. 돈이 샌 게 아닙니다 |

> 💡 **m2net 을 거친 통화·채팅은 전부 실제 과금입니다. "테스트 통화/가짜 결제" 같은 건 없습니다.** (내가 내 돈으로 직접 써본 것도 실사용이지 테스트가 아님)

> 🔒 돈 관련 숫자(상담시간·수익금·코인)가 의심되면 개발 측에 **`python tools/_verify_money_integrity.py`** 실행을 요청하세요. 5대 돈 불변식을 자동 점검해 "이상 없음/사고"를 객관적으로 알려줍니다.

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 회원 m2net 등록 상태 | 사이드바 → 회원현황 → 고객 리스트 → 상세 |
| 상담사 m2net 등록 상태 | 사이드바 → 회원현황 → 상담사 리스트 → 상세 |
| 통화·채팅 이력 | 사이드바 → 매출현황 → 사용(상담) 내역 |
| 잔액 정정 (수동) | 사이드바 → 매출현황 → 포인트 관리 |

## m2net 측 확인 (사장님 직접)

사주플랜 시스템에서 못 보는 정보 (m2net DB) 가 있을 때:
- m2net 콘솔에 사장님 영업담당 계정으로 접근
- csrid (상담사 ID) / 등록 phone / 통화 이력 등 확인 가능
