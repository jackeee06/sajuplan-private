# 사주플랜페이 자동충전

## 한 줄로 답하면
**회원이 카드를 한 번 등록하면, 코인 잔액이 부족할 때 사주플랜이 자동으로 결제해 코인을 충전합니다. 상담 중 갑자기 코인이 부족해 끊기는 상황을 막기 위한 기능입니다.**

---

## 왜 자동충전을 만들었나요?

1. **상담 중 끊김 방지** — 전화·채팅 상담 중 코인이 소진되면 상담이 강제 종료됩니다. 이때 회원이 즉시 충전하기 어렵고, 상담사도 갑작스럽게 상담이 끊깁니다. 자동충전이 있으면 이 문제가 없어집니다.
2. **회원 편의** — 매번 충전 페이지를 방문하는 번거로움을 없애줍니다.
3. **플랫폼 매출 안정화** — 코인 부족으로 상담을 못 하는 경우를 줄여 플랫폼 전체 매출이 늘어납니다.

---

## 동작 흐름

```
회원이 카드 등록 (사주플랜페이)
  ↓
상담 진행 중 코인 잔액이 임계값 이하 감지
  ↓
사주플랜이 등록된 카드로 자동 결제 (AG9 빌링)
  ↓
결제 성공 → 코인 즉시 적립
  ↓
회원에게 알림톡 발송 (자동충전 완료 안내)
```

---

## 카드 등록 흐름

```
회원 → 마이페이지 → 충전 → 자동충전 카드 등록
  ↓
AG9에서 빌링키 발급 (카드번호 직접 저장 X, 토큰만 저장)
  ↓
DB에 빌링키 저장 (payment_autopay 테이블)
```

> ⚠️ 카드번호는 사주플랜 서버에 저장되지 않습니다. AG9(PG사)에서 발급한 빌링키만 저장합니다. 보안 원칙.

---

## 경우의 수별 설명

### Q. "자동충전 등록 현황을 보려면?"
→ 매출현황 → 결제 내역 → `자동충전` 탭 필터

### Q. "자동충전이 실패하면?"
→ 결제 실패 알림톡이 회원에게 발송됩니다.
→ 실패 이력은 결제 내역에서 확인 가능

### Q. "회원이 카드를 삭제하면?"
→ 자동충전 비활성화. 다음 상담 전에 회원이 직접 재등록해야 합니다.

### Q. "자동충전 금액은 어떻게 결정되나요?"
→ 충전금액 설정에 등록된 상품 중 특정 금액으로 자동 설정됩니다.

---

## 현재 운영 상태

현재 실 등록 건수: 코드 완비, 실 사용 테스트 중.
자동충전 관련 이슈 발생 시 매출현황 → 결제 내역에서 확인.

---

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 |
|------|------|
| 상담 중 자동 결제 후 코인 적립 | ✅ 정상 |
| 카드 한도 초과로 자동충전 실패 | ✅ 정상 (회원에게 알림) |
| 빌링키 없이 자동충전 시도 | ❌ 버그 — 즉시 확인 |
| 자동충전 후 중복 코인 적립 | ❌ 이중충전 버그 — 즉시 확인 |

---

## 운영자가 확인하는 위치

| 작업 | 메뉴 위치 |
|------|---------|
| 자동충전 등록 현황 | 매출현황 → 결제 내역 (자동충전 필터) |
| 자동충전 실패 이력 | 매출현황 → 결제 내역 (실패 상태 필터) |
