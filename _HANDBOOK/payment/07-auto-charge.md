# 자동충전 (사주플랜페이)

## 한 줄로 답하면
**회원이 카드 1회 등록하면 잔액이 부족할 때 사주플랜이 자동으로 결제 → 코인 적립. 운영 시작 후 활성화 예정.**

## 동작

```
회원이 충전 페이지 → 자동충전 등록 → 카드 정보 입력
  ↓
사주플랜이 카드사 (AG9 PG) 에 빌링키 발급 요청
  ↓
빌링키 등록 완료 → 회원에게 안내
  ↓
[회원이 채팅·통화 사용 → 잔액 부족]
  ↓
사주플랜이 자동 결제 (등록된 카드로)
  ↓
결제 성공 → 코인 적립 → 회원에게 알림톡
```

## 현재 상태

| 항목 | 상태 |
|---|---|
| 코드 완성 | ✅ 가능 |
| 실 카드 등록 | ❌ 운영 시작 전 0건 |
| 운영 시작 후 | 회원이 등록하면 자동 활성화 |

## 이런 상황을 보셨나요?

### 회원이 "자동충전 어떻게 설정해요?" 라고 문의

→ 회원 마이페이지 → 충전 → 자동충전 등록 → 카드 정보 입력 → 완료. 다음 사용 시점부터 잔액 부족 시 자동 결제.

### "카드 등록했는데 에러 떴어요"

→ AG9 PG 측 에러 (한도, 카드 정보, 만료 등). 회원에게 안내 + 다른 카드 등록 권장.

### "자동 결제 됐는데 코인 안 들어왔어요"

→ 일반 충전과 동일 흐름. 자동 정정 시스템이 처리. 5분 이상 지연 시 운영자 점검.

### "자동충전 끄고 싶어요"

→ 회원 마이페이지 → 충전 → 자동충전 → 해제. 다음부터 자동 결제 X.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 회원이 카드 등록 → 빌링키 발급 성공 | ✅ 정상 |
| 잔액 부족 시 자동 결제 + 코인 적립 | ✅ 정상 |
| 자동 결제 실패 (카드 만료) → 알림톡 + 재시도 | ✅ 정상 (안전망) |
| 카드 미등록 회원이 자동 결제됨 | ❌ **사고**. 즉시 점검 |
| 같은 시간에 여러 번 자동 결제 | ❌ **사고**. 중복 방지 점검 |

## 운영 시작 시 주의

운영 시작 직후 자동충전 첫 등록자 발생 시:
- 운영자가 모니터링 권장
- 카드 등록 / 자동 결제 흐름 정상 동작 확인
- 첫 케이스 사고 시 메모리/매뉴얼에 박제

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 자동충전 등록 회원 (목록) | 사이드바 → 회원현황 → 고객 리스트 → 필터 "자동충전" |
| 자동 결제 이력 | 사이드바 → 매출현황 → 결제 내역 → 결제 방법 "자동" |
| 자동 결제 실패 케이스 | 사이드바 → 매출현황 → 결제 내역 → 상태 "실패" |
