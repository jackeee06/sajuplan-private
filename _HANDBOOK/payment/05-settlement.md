# 상담사 정산은 어떻게 이뤄지나요?

## 한 줄로 답하면
**상담 1건 끝나면 상담사 수익금이 누적 → 매월 말 자동 정산 cron 이 등급별 단가 적용 + 원천세 3.3% 차감 후 통장 입금 준비.**

## 정산 흐름

```
회원 채팅·통화 완료
  ↓
상담사 수익금 적립 (earning_balance)
  ↓
매월 말 자동 정산:
  - 상담사별 누적 earning_balance 집계
  - 등급별 마진 비율 적용
  - 원천세 3.3% 차감 (소득세 3% + 지방세 0.3%)
  - 정산 row 생성
  ↓
운영자 (사장님) 검토
  ↓
통장 입금
```

## 이런 상황을 보셨나요?

### 상담사가 "이번 달 정산 언제 들어와요?" 라고 문의

→ 매월 말 자동 정산 → 사장님 검토 → 통장 입금. 보통 다음 달 초.

→ 답변: "매월 말 정산 후 운영팀 확인 거쳐 다음 달 초 통장 입금됩니다."

### 상담사가 "환불 발생했는데 내 수익금이 줄어요" 라고 문의

→ 정상. 회원이 환불 받으면 그 채팅의 상담사 수익도 같이 차감.

→ 답변: "환불은 해당 상담의 수익이 같이 차감되는 정책입니다."

### "정산 금액이 예상보다 적어요"

→ 원천세 3.3% (소득세 + 지방세) 자동 차감되어서. 누적 수익금 × (1 - 0.033) = 실 입금.

## 원천세 (3.3%)

- 소득세 3% + 지방세 0.3%
- 사업소득 원천징수 (한국 세법)
- 매월 정산 시 자동 차감
- 분기별 세무서 신고 (운영자 작업)

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 상담 완료 → earning_balance 증가 | ✅ 정상 |
| 매월 말 자동 정산 row 생성 | ✅ 정상 |
| 환불 발생 → 해당 상담사 earning 차감 | ✅ 정상 |
| 정산 금액에 원천세 자동 차감 | ✅ 정상 |
| 정산 row 안 만들어짐 | ❌ **사고**. 정산 cron 점검 |
| earning_balance 가 음수 | ⚠️ 환불 다수로 음수 가능. 검토 필요 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 정산 이력 (전체) | 사이드바 → 매출현황 → 정산 이력 |
| 상담사별 누적 수익금 | 사이드바 → 회원현황 → 상담사 리스트 → 상세 |
| 매월 정산 실행 | 자동 (월말 cron) — 수동 트리거 가능 |
| 수동 정산 (특수 케이스) | 사이드바 → 매출현황 → 정산 이력 → 등록 |
