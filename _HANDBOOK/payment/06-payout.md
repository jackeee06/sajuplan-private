# 상담사가 정산 전에 미리 받고 싶어요 (선지급)

## 한 줄로 답하면
**상담사가 매월 정산 전에 누적 수익금의 최대 70% 까지 미리 받을 수 있습니다. 수수료 5% + 원천세 3.3% 차감 후 통장 입금. 하루 1회 신청 가능.**

## 정책

| 항목 | 값 |
|---|---|
| 가용 한도 | 누적 수익금의 **70%** |
| 수수료 | **5%** |
| 원천징수 | **3.3%** (정산과 동일) |
| 신청 한도 | **하루 1회** |

## 계산 예시

상담사 누적 수익금 1,000,000원, 선지급 신청:

```
1. 가용 한도 = 1,000,000 × 70% = 700,000원
2. 수수료 = 700,000 × 5% = 35,000원
3. 원천세 = 700,000 × 3.3% = 23,100원
4. 실 입금 = 700,000 - 35,000 - 23,100 = 641,900원
5. 다음 정산에서 700,000원이 자동 차감 (이미 받은 셈)
```

## 신청 흐름

```
상담사가 마이페이지 → 선지급 신청
  ↓
신청 알림톡 도착 ("선지급 신청 접수")
  ↓
운영자 (사장님) 검토 — 사이드바 → 매출현황 → 선지급 관리
  ↓
승인 → 통장 입금 → "선지급 입금 완료" 알림톡
반려 → 사유 입력 → "선지급 반려 안내" 알림톡
```

## 이런 상황을 보셨나요?

### 상담사가 "선지급 신청했는데 즉시 입금 안 됐어요" 라고 문의

→ 정상. 운영자 검토 + 영업시간 내 처리.

→ 답변: "운영팀 확인 후 영업시간 내 처리됩니다."

### 상담사가 "70% 한도 넘게 신청하고 싶어요"

→ 정책상 불가. 30% 는 환불 발생 시 회사 흡수 버퍼.

→ 답변: "선지급은 누적 수익금의 70% 까지만 가능합니다. 100% 는 매월 말 정산에서 받으실 수 있습니다."

### "같은 날 두 번 신청 가능한가요?"

→ 하루 1회 한도. 두 번째는 자동 거부.

### "환불 발생 후 가용 한도 줄었어요"

→ 정상. 환불 = 누적 수익금 감소 → 가용 한도 (70%) 도 감소.

## 정산 vs 선지급 차이

| 항목 | 정산 (매월 말) | 선지급 (상시) |
|---|---|---|
| 빈도 | 매월 1회 자동 | 매일 1회 가능 (상담사 신청) |
| 한도 | 100% (전액) | 누적 70% |
| 수수료 | 없음 | 5% |
| 처리 | 자동 | 운영자 수동 검토 |
| 영업일 | 매월 초 | 영업시간 내 |

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 누적 100만원 → 70만원 신청 가능 | ✅ 정상 |
| 같은 날 2번 신청 → 두 번째 거부 | ✅ 정상 |
| 신청 후 즉시 입금 안 됨 | ✅ 정상 (운영자 검토 필요) |
| 다음 정산 시 선지급 자동 차감 | ✅ 정상 |
| 다음 정산에서 차감 안 됨 | ❌ **사고**. 자동 차감 로직 점검 |
| 가용 한도가 음수 | ⚠️ 환불 다수로 발생 가능. 신청 거부 |

## 운영자가 확인하는 위치

| 확인 항목 | 메뉴 위치 |
|---|---|
| 선지급 신청 목록 | 사이드바 → 매출현황 → 선지급 관리 |
| 승인 / 반려 처리 | 사이드바 → 매출현황 → 선지급 관리 → 상세 |
| 선지급 이력 | 사이드바 → 매출현황 → 선지급 관리 → 이력 탭 |
