# 상담사 선지급 정책 (수수료·가용·한도)

## 한 줄로 답하면
**상담사가 매월 정산 전에 현재 자기 수익금(earning_balance)의 최대 70%를 미리 받을 수 있습니다. 수수료 5% + 원천세 3.3% 차감 후 통장 입금. 하루 1회 · 최소 3만원부터 신청 가능.**

> **현재 상태 (2026-06-10)**: 시스템은 준비됐으나 **prod 실제 선지급 신청 0건** (아직 아무도 사용 안 함).

---

## 왜 선지급 시스템을 만들었나요?

1. **상담사 현금흐름 개선** — 정산은 매월 1일에만 이루어집니다. 중순에 돈이 급한 상담사가 한 달을 꼬박 기다려야 하는 불편을 해소합니다.
2. **플랫폼 충성도 강화** — 급할 때 당겨받을 수 있다는 안전망이 상담사 이탈을 줄입니다.
3. **70% 한도 이유** — 전액 미리 주면 정산일에 환불·공제 등으로 음수가 발생할 수 있습니다. 30%는 버퍼로 남깁니다.
4. **수수료 5% 이유** — 미리 지급하는 데 따른 유동성 비용. 사주플랜이 먼저 자금을 집행하는 금융 서비스 성격이 있기 때문입니다.

---

## 선지급 정책 (현재 설정값)

| 항목 | 값 | 설명 |
|------|------|------|
| 가용 한도 | **현재 수익금(earning_balance)의 70%** | 나머지 30%는 버퍼 (환불·공제 대비) |
| 수수료 | **5%** | 선지급 금액에서 차감 |
| 원천세 | **3.3%** | 공급가 기준 법적 원천징수 |
| 일 신청 횟수 | **1회** | 하루 1회만 신청 가능 |
| 최소 금액 | **30,000원** | 3만원 미만은 신청 불가 |

> **가용 한도 기준 = "지금 그 사람 수익금(earning_balance)의 70%"** 입니다. 과거 누적 전체가 아니라, 이미 정산·선지급으로 빠진 것을 뺀 **현재 남은 수익금**이 기준입니다.

---

## 실수령 계산 예시

상담사가 이번 달 누적 수익금 100만원, 선지급 50만원 신청 시:
```
신청금액:       500,000원
수수료 5%:      -25,000원
원천세 3.3%:    -15,675원 (= (500,000 - 25,000) × 0.033)
──────────────────────────
실수령:         459,325원
```

## 경우의 수별 설명

### Q. "상담사가 선지급을 신청했어요 — 어디서 처리하나요?"
→ 매출현황 → 선지급 관리 → **처리 대기** 탭 확인

### Q. "선지급 처리 절차는?"
```
1. 선지급 관리 → 처리 대기 탭
2. 해당 건 확인 (신청금·수수료·실지급액 확인)
3. CSV 다운로드 → 은행 일괄이체 실행
4. 입금 완료 후 "지급완료" 처리
5. 상담사에게 알림톡 자동 발송
```

### Q. "선지급 반려 시 사유를 써야 하나요?"
→ 네. 반려 시 사유 입력 필수. 상담사에게 알림톡으로 자동 전달됩니다.

### Q. "24시간 이상 미처리 경고가 뜨는데?"
→ 선지급 관리 상단에 빨간 경고 표시. 즉시 처리하거나 반려 처리해야 합니다.

### Q. "선지급 후 정산일에 어떻게 되나요?"
→ **아직 정산 안 된 선지급 금액(`status='paid'` AND 정산처리 전)은 정산예상금액에서 자동으로 빠집니다.** 이미 미리 받은 돈이니 정산 때 또 주면 이중 지급이 되기 때문입니다.
→ 정산을 [정산하기] 버튼으로 실제 처리하면, 해당 선지급 건에 `settled_at`(정산완료 시각)이 찍히면서 "정산 반영됨"으로 마킹됩니다. 다음 달 정산부터는 그 선지급은 더 이상 차감 대상이 아닙니다.
→ (정산 단순화 전환 작업에서 이 연동을 구현 예정 — `payment/05-settlement` 참조)

### Q. "이번 달 수익금보다 많이 선지급 신청하면?"
→ 시스템에서 가용 한도(70%) 초과 시 신청 자체가 불가합니다.

## 이건 정상인가요? 에러인가요?

| 상황 | 판단 |
|------|------|
| 수수료 5% 차감 후 입금 | ✅ 정상 |
| 원천세 3.3% 차감 후 입금 | ✅ 정상 |
| 하루 2번 신청 불가 | ✅ 정상 (일 1회 한도) |
| 70% 이상 신청 불가 | ✅ 정상 (가용 한도) |
| 지급 완료인데 수익금(earning_balance)이 그대로 | ✅ 정상 (수익금 차감은 정산 [정산하기] 처리 시점에 발생) |
| 정산예상금액이 선지급만큼 적게 나옴 | ✅ 정상 (미정산 선지급분은 정산에서 제외 → 이중 지급 방지) |
| 24시간 후에도 처리 대기 중 | ⚠️ 즉시 처리 필요 |

## 운영자가 확인하는 위치

| 작업 | 메뉴 위치 |
|------|---------|
| 선지급 신청 처리 | 매출현황 → 선지급 관리 |
| 수수료율·원천세율 변경 | 환경설정 → 기본환경설정 → 선지급 탭 (슈퍼 전용) |
| 정책 안내문 수정 | 환경설정 → 기본환경설정 → 선지급 탭 |
