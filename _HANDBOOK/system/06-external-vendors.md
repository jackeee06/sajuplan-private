# 외부 서비스 협의 (BizM·AG9·m2net·FCM)

## 한 줄로 답하면
**사주플랜은 4개 외부 서비스 의존. 각 서비스별 협의 채널·결제 방식·사고 대응 방법 다름.**

## 외부 서비스 4종

| 서비스 | 역할 | 의존도 |
|---|---|---|
| m2net (PassCall) | 통화·채팅 라우팅, 가상번호, 잔액 | 🔴 매우 큼 |
| AG9 (PG) | 카드·가상계좌·간편결제 | 🔴 매우 큼 (매출) |
| BizM | 카카오 알림톡 | 🟡 (안내) |
| Firebase FCM | 모바일 푸시 | 🟢 (거의 안 씀) |

## 협의 채널

| 서비스 | 협의 방법 |
|---|---|
| m2net | 사장님 영업담당 (직접) |
| AG9 | 가맹점 등록, AG9 고객센터 |
| BizM | BizM 콘솔 + 고객센터 |
| Firebase | Firebase Console + 일반 문서 |

## 비용 (월 추정)

| 서비스 | 비용 |
|---|---|
| m2net | 사용량 기반 (가입 + 통화/채팅 단가) |
| AG9 | 결제 건당 수수료 (약 3-4%) |
| BizM | 알림톡 건당 약 7원 |
| FCM | 무료 |
| **합계** | 매출의 약 5-10% (초기) |

## 이런 상황을 보셨나요?

### "통화·채팅 안 돼요" 다수 클레임

→ m2net 측 사고 의심. 즉시 사장님 영업담당 연락. m2net 측 상태 확인.

### "결제 안 됨" 다수 클레임

→ AG9 PG 측 사고 의심. 결제 페이지 일시 차단 검토. AG9 고객센터 연락.

### "알림톡 못 받았어요" 다수 클레임

→ BizM 측 사고 또는 가맹점 발신 차단. BizM 콘솔 확인.

### "푸시 안 와요"

→ FCM 측 사고 거의 없음. 회원 폰 알림 권한 / 사주플랜 앱 미설치 의심.

## 외부 의존성 사고 대응

| 사고 | 1차 조치 |
|---|---|
| m2net 다운 | 사장님 영업담당 즉시 연락. 회원에게 일시적 안내 |
| AG9 다운 | 결제 페이지 일시 차단 검토. AG9 고객센터 |
| BizM 다운 | 사용자에게 인앱 알림은 정상. 알림톡만 일시 중단 |
| FCM 다운 | 거의 영향 없음 |

## 외부 서비스 변경 시 (예: m2net → 다른 통신사)

대공사. 신중한 검토 필요:
- API 포맷 / callback 흐름 재구현
- 회원·상담사 정보 마이그레이션
- 기간 중 양 시스템 병행 운영
- 사장님 결정 + 충분한 사전 준비
