# 배포는 어떻게 이뤄지나요?

## 한 줄로 답하면
**모든 코드 변경은 사장님 검토 후 운영 서버(PROD) 배포가 표준. TEST 서버는 2026-05-29 폐기. md 같은 작은 변경은 외과 배포 (빠르게).**

## 배포 흐름

```
코드 변경
  ↓
빌드 (Vite/TypeScript)
  ↓
양 서버 (PROD + TEST) SFTP 동시 배포
  ↓
환경 치환 (prod/test 구분)
  ↓
사용자 화면 즉시 반영
```

## 운영자 입장

실제 코드 배포는 시스템이 처리. 운영자가 직접 배포하지 않음. **사장님이나 개발자**가 변경 후 배포 명령 실행.

## 이런 상황을 보셨나요?

### "기능 수정 요청했는데 언제 반영돼요?"

→ 코드 수정 → 검토 → 배포까지 한 세트. 일반적으로 빠르게 처리.

### "배포 직후 회원에게 옛 화면 보여요"

→ 브라우저 캐시. 강제 새로고침 (Ctrl+Shift+R) 또는 시간 지나면 자동 갱신.

### "TEST 환경 접속 가능해요?"

→ sajumoon.kr (테스트 서버). 일반 회원에게는 안내 X. 사장님/개발자/QA 만.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 배포 후 즉시 사용자 화면 반영 | ✅ 정상 |
| 배포 후 사용자 캐시 때문에 옛 화면 | ⚠️ 일시적. 시간 지나면 갱신 |
| 배포 후 prod 에서 test API 호출 사고 | ❌ 환경 치환 누락. 즉시 재배포 |
| TEST 서버 (sajumoon.kr) 다운 | ⚪ 폐기된 서버 — 대응 불필요 |
