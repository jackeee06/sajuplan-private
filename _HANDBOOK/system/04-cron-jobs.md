# 사주플랜 자동 작업 (cron)

## 한 줄로 답하면
**사주플랜이 자동으로 매분/시간/일 단위로 정산·자동취소·재시도·통계 처리. 운영자가 별도 트리거 안 해도 알아서.**

## 주요 자동 작업

| 작업 | 주기 | 동작 |
|---|---|---|
| 채팅 3분 자동 취소 | 매분 | 응답 없는 채팅 정리 + 알림톡 |
| m2net 동기화 재시도 | 10분 | 옛 동기화 실패 건 재시도 |
| 5분/30초 잔여 알림 | 매분 (백업) | 클라이언트 폴링 누락 보완 |
| 상담사 자동 부재중 | 1시간 | 24시간 무응답 상담사 전환 |
| 매월 말 정산 | 매월 | 상담사 누적 수익금 정산 |
| 매일 09:00 일일 요약 | 매일 09:00 | 사장님에게 운영 요약 |
| 외부 의존성 헬스 체크 | 5분 | DB / m2net / BizM 상태 |

## 이런 상황을 보셨나요?

### "STAY 채팅방이 5분 넘게 살아있어요"

→ 3분 자동 취소 cron 정지 의심. 즉시 점검 필요.

### "매일 일일 요약 알림 안 와요"

→ 가능 원인: cron 정지 / BizM 실패 / 사장님 채팅 중 차단.

### "매월 정산 누락됐어요"

→ 정산 cron 정지 가능. 수동 정산 (사이드바 → 매출현황 → 정산 이력 → 등록) 으로 처리 후 cron 원인 점검.

## cron 정지 시 영향

| 정지 | 영향 |
|---|---|
| 3분 자동 취소 | 응답 없는 채팅 무한 대기 |
| m2net 동기화 재시도 | 잔액 어긋남 누적 |
| 정산 cron | 상담사 입금 지연 |
| 일일 요약 | 사장님이 매출 모름 |
| 헬스 체크 | 외부 의존성 사고 늦게 인지 |

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 매분 자동 취소 로그 | ✅ 정상 |
| 매월 1일 정산 row 생성 | ✅ 정상 |
| 매일 09:00 일일 요약 도착 | ✅ 정상 |
| STAY 채팅방 4분+ 유지 | ❌ cron 정지. 점검 |
| 정산 누락 (월말 기준) | ❌ cron 정지 또는 데이터 사고 |
