# 사주플랜 앱 (WebView) 의 특수 동작

## 한 줄로 답하면
**사주플랜 앱은 RN WebView 셸 (앱 안에 사주플랜 웹 사이트). 일반 모바일 웹과 다른 특이점 몇 가지 있음 — 외부 URL 차단·캐시 갱신 지연 등.**

## 주요 특이점

### 외부 URL 차단
- 앱 WebView 안에서 외부 사이트 클릭하면 새 탭 안 열림
- "openExternalUrl" 같은 별도 처리 필요 (사주플랜 시스템이 알아서)

### 캐시 갱신 지연
- 앱이 옛 화면 캐시하고 있을 수 있음
- 새 정책 배포 후 일부 사용자에게 옛 화면 보이는 경우 가능
- 강제 새로고침 또는 시간 지나면 갱신

### 알림 → 앱 자동 호출 (APK 등록 필요)
- 알림톡 클릭 시 사주플랜 앱 자동으로 열려야 함
- APK 측 설정 + BizM 알림톡 URL 변경 필요 (현재 진행 중)

## 이런 상황을 보셨나요?

### "회원이 옛 화면 보고 있대요"

→ 캐시 가능성 1순위. 강제 새로고침 안내. 그래도 안 되면 시스템 사고.

### "외부 링크 클릭이 안 돼요"

→ WebView 한계. 사주플랜 화면에서 외부 사이트는 별도 처리 필요. 운영자가 등록한 외부 링크 (배너, 공지 등) 가 안 되면 시스템 점검.

### "알림톡 클릭하니 카톡 브라우저에서 사주플랜 사이트가 열려요"

→ 현재 APK 미등록 상태라 그런 것. APK 정식 등록되면 앱 자동 호출. 별도 트랙으로 작업 중.

## 이건 정상인가요? 에러인가요?

| 상황 | 답변 |
|---|---|
| 앱 안에서 사주플랜 동작 | ✅ 정상 |
| 외부 사이트 클릭 → 시스템 브라우저 열림 | ✅ 정상 (sajuplan 자동 처리) |
| 옛 캐시로 옛 화면 노출 | ⚠️ 일시적 |
| 알림톡 클릭 → 카톡 브라우저에서 사주플랜 사이트 | ⚠️ APK 등록 전 (별도 트랙) |
| 회원이 앱 못 열어요 | ❌ APK 다운로드 / 설치 안내 |
