# 배포 후 새 기능 확인 방법

## 핵심 원칙

JS/CSS 파일은 Vite 빌드 시 파일명에 해시가 자동으로 붙습니다.
예: `index-txRNRlym.js` → 변경되면 `index-AbCdEfGh.js`로 이름이 바뀜.
→ 파일 자체는 자동으로 새로 로드됩니다.

**문제는 index.html** — 이 파일이 캐시되면 옛날 JS/CSS를 계속 불러옵니다.

---

## 환경별 확인 방법

### 브라우저 (PC/맥)

**방법 1: 강제 새로고침** (99% 해결)
- Windows: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

**방법 2: 그래도 안 되면 — 캐시 삭제**
- 개발자도구(F12) → Network 탭 → "Disable cache" 체크 → 새로고침

---

### 앱 (안드로이드 / iOS WebView)

앱은 브라우저보다 캐시가 강합니다.

**1단계: 앱 완전 종료 후 재시작** (먼저 해보기)
- 안드로이드: 최근 앱 목록 → 사주플랜 위로 밀어서 종료 → 재실행
- iOS: 앱 스위처 → 사주플랜 위로 밀어서 종료 → 재실행

**2단계: 여전히 안 보이면 — 앱 데이터/캐시 삭제**
- 안드로이드: 설정 → 앱 → 사주플랜 → 저장소 → 캐시 삭제
- iOS: 앱 삭제 후 재설치 (iOS는 캐시만 삭제 불가)

**쿠키(로그인 정보)는 삭제 안 해도 됩니다.** 캐시만 삭제하면 됩니다.

---

## 어디서 확인하나

| 기능 | 확인 위치 |
|---|---|
| 메인/홈 화면 | 앱 실행 후 홈 탭 |
| 상담사 문의 탭 | 상담사 탭 → 상담사 선택 → 문의 탭 |
| 나의 상담문의 | 마이 탭 → 상담문의 |
| 관리자 화면 | 앱/브라우저에서 `sajuplan.com/mng` |

---

## 언제 쿠키를 지워야 하나

거의 없음. 아래 경우에만:
- 로그인이 이상하게 유지될 때
- 권한이 바뀌었는데 반영 안 될 때 (로그아웃 → 재로그인으로 해결)

---

## 결론

대부분: **앱 완전 종료 → 재시작** 으로 해결
안 될 때: **캐시 삭제** (쿠키는 건드리지 않음)

## Claude의 완료 보고 형식 (배포 후 항상 이 형식으로)

```
요청하신 작업: (한 줄 요약)
수행한 작업:
  - 변경한 파일 + 무엇을 바꿨는지
  - 어떤 기능이 추가/수정됐는지
확인 방법: 앱 완전 종료 → 재시작 → [어디서 보는지]
```

예시:
```
요청하신 작업: 문의 탭 디자인 개선

수행한 작업:
  - CounselorQna.tsx: 배지 제거, 작성자+날짜 한 줄, 비밀글 통합 표시
  - CounselorQnaDetail.tsx: 본인 글 수정/삭제 버튼 추가

확인 방법: 앱 종료 후 재시작 → 상담사 탭 → 상담사 선택 → 문의 탭
```
