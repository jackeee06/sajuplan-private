# 사주플랜 쿠폰 & 무료코인 정책 (2026-06-07 확정)

> **이 문서가 유일한 기준.** 코드·DB·관리자 화면이 이 문서와 어긋나면 이 문서가 맞다.

---

## 핵심 원칙

**쿠폰과 코인 지급은 절대 중복되지 않는다.**
- 같은 이벤트에 쿠폰 + 코인 동시 지급 금지
- 각 이벤트는 둘 중 하나의 방식만 사용

---

## 무료코인 지급 (즉시 free_balance 적립 — 쿠폰함 미노출)

| 이벤트 | 지급 방식 | 금액 | 설정 위치 |
|---|---|---|---|
| **회원가입** | 즉시 코인 (`creditRegisterPoint`) | `setting.member.register_point` 값 | 관리자 → 환경설정 |
| **로그인** | 즉시 코인, 하루 1회 (`creditLoginPointOnce`) | `setting.member.login_point` 값 | 관리자 → 환경설정 |
| **출석 1일** | 즉시 코인 | `setting.attendance.user.day1` | 관리자 → 환경설정 |
| **출석 5/10/15/20일 보너스** | 즉시 코인 추가 | 각 setting 값 | 관리자 → 환경설정 |
| **출석 30일 달성** | 즉시 코인 (쿠폰 전환 예정이나 현재 코인) | `setting.attendance.user.day30_coupon_amount` | 관리자 → 환경설정 |
| **후기 작성** | 즉시 코인 | 관리자 설정값 | 관리자 → 환경설정 |
| **베스트 후기 선정** | 즉시 코인 10,000 | 고정 | 관리자 → 후기관리 |

> 위 항목은 모두 `point_history`에 `rel_action` 으로 기록되며 m2net에 자동 동기화됨.

---

## 쿠폰 (쿠폰함 경유 — 회원이 직접 "사용" 버튼 클릭 시 코인 전환)

| 종류 | 발급 방법 | 비고 |
|---|---|---|
| **코드입력 쿠폰** (cz_type=3) | 회원이 쿠폰함 → 코드 입력 | 코드 입력 즉시 사용+코인 지급. 현재 미운영 |
| **관리자 수동 발급** | 관리자 → 쿠폰존관리에서 특정 회원에게 직접 발급 | 이름을 반드시 사용자 친화적으로 작성할 것 |

## 폐지된 쿠폰 (2026-06-07)

| 항목 | 폐지 사유 |
|---|---|
| **회원가입 쿠폰** (`issueSignupCoupon`) | `creditRegisterPoint`(즉시 코인)와 이중지급 위험. 코인 직접지급으로 통일 |

- DB: `coupon_zone` id=1 `is_active=false`
- DB: 기발급된 미사용 쿠폰 17건 `hidden_at` 마킹 (회원 쿠폰함에서 숨김)
- 코드: `auth.service.ts` `createLocalMember()` 에서 `issueSignupCoupon()` 호출 제거

---

## 관리자 수동 쿠폰 발급 시 주의사항

- **쿠폰 이름**은 반드시 사용자 화면에 노출되는 이름으로 작성 (예: "서비스 혜택 코인", "운영팀 지급")
- 내부 개발 용어 사용 금지 (예: "자동sync검증쿠폰" ← 이런 거 금지)
- 발급 전 동일 회원에게 이미 지급된 코인/쿠폰이 있는지 확인

---

## 쿠폰함 화면 설명

- **쿠폰함** 탭: 사용 가능한 쿠폰 목록 (관리자 수동 발급된 것)
- **사용내역** 탭: 사용 완료된 쿠폰 히스토리
- 출석/후기/회원가입 보상은 즉시 코인이라 **쿠폰함에 절대 나타나지 않음**
